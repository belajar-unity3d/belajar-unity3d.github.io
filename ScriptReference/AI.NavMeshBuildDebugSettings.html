Description

Tentukan data sementara yang dihasilkan saat membangun NavMesh harus dipertahankan dalam memori setelah proses selesai.

Mungkin untuk mengumpulkan dan menampilkan di Editor data menengah yang digunakan dalam proses membangun jaring navigasi menggunakan NavMeshBuilder. Ini dapat membantu dengan mendiagnosis situasi tersebut ketika dihasilkan NavMesh bukan bentuk yang diharapkan.

Masukan Geometry, Wilayah, Polygonal Detail Mesh dan Kontur Mentah ditampilkan setelah membangun NavMesh dengan opsi debug




Proses untuk komputasi NavMesh terdiri dari beberapa langkah sequential:
i. dekompos medan Adegan dan jala menjadi segitiga;
ii. rasterizing segitiga input menjadi representasi voxel 3D dan menemukan ledges;
iii. partisi voxels berbaring di permukaan ke daerah horizontal sederhana;
iv. menemukan kontur ketat untuk setiap daerah ini;



Depending pada komposisi adegan data debug ini dapat sangat besar dalam ukuran. Hal ini disimpan dalam memori dengan cara terkompresi tetapi semakin diperluas ketika ditampilkan.

Notes:

  1. Unity tidak menyimpan visualisasi Debug - mereka hanya tersedia selama sesi di Unity adalah membangun NavMesh.
  2. Data debug tidak ditampilkan atau dikumpulkan untuk NavMeshes yang secara statis dipanggang dengan sistem legacy, atau untuk setiap patch yang dikomputasikan secara lokal karena kehadiran NavMesh Obstacles.

Lihat Juga: NavMeshBuildSettings, NavMeshBuilder.BuildNavMeshData, NavMeshEditorHelpers.DrawBuildDebug.
