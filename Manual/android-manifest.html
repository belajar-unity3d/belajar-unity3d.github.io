Android SDK

Android App Manifest berisi informasi tentang aplikasi Android. Setiap aplikasi memiliki satu aplikasi Android App Manifest XML file di akar set sumber disebut AndroidManifest.xml. Sistem operasi Android dan layanan distribusi digital (misalnya, Google Play) menggunakan Android App Manifests untuk menemukan informasi, seperti nama aplikasi, titik masuk aplikasi, dukungan versi Android, dukungan fitur perangkat keras, dan izin aplikasi. Untuk informasi lebih lanjut tentang file Manifest Aplikasi Android, dan untuk daftar pengaturan yang dikonfigurasi, lihat dokumentasi Pengembang Android pada Android SDK.

Untuk menghasilkan Android App Manifest untuk mewakili aplikasi, Gradle menggabungkan file manifest dari berbagai sumber. Ini termasuk:

  • Unity Library Manifest: manifestasi file yang Unity menghasilkan yang mengkonfigurasi kegiatan Unity Player. Untuk informasi lebih lanjut, lihat Manifest Perpustakaan Unity.
  • Unity Launcher Manifest: manifestasi file yang Unity menghasilkan yang mengkonfigurasi aplikasi yang membungkus perpustakaan Unity. Untuk informasi lebih lanjut, lihat Unity Peluncur Manifest.
  • Plug-inA set of code created outside of Unity that creates functionality in Unity. There are two kinds of plug-ins you can use in Unity: Managed plug-ins (managed .NET assemblies created with tools like Visual Studio) and Native plug-ins (platform-specific native code libraries). More info
    See in Glossary     manifests    : file Manifest yang mewakili plug-ins seperti Arsip Android (AAR) atau Proyek Perpustakaan Android.

Untuk informasi tentang bagaimana Unity menggunakan file manifest ini untuk menghasilkan Android App Manifest, lihat Menghasilkan Android App Manifest.

Menghasilkan Android App Manifest

Proses build aplikasi Android menghasilkan file Manifest aplikasi Android untuk aplikasi. Untuk melakukan ini:

  1. Unity menggunakan Manifest Perpustakaan Unity sebagai template untuk Android App Manifest. Jika Anda menimpa Manifest Perpustakaan Unity, Unity menggunakan file yang Anda tentukan sebagai template.
  2. Pembaruan Unity Perpustakaan Manifest and Unity Launcher Manifest file dengan informasi seperti permissions, opsi konfigurasi, dan fitur yang digunakan aplikasi.
  3. GradleSistem build Android yang mengotomatiskan beberapa proses build. Otomasi ini berarti bahwa banyak kesalahan build umum cenderung terjadi. More info
    Lihat di Glossary     menggabungkan Manifest Perpustakaan Unity, Manifest Peluncur Unity, dan manifestasi plug-in menjadi satu file Manifest Aplikasi Android.

Anda dapat melihat file Android App Manifest di dalam output Android App Bundle (AAB) atau Paket Android (APK) menggunakan Android APK, atau alat pihak ketiga lainnya seperti Apktool.

Important: Anda tidak dapat mengedit file Manifest Android di APKFormat paket Android output oleh Unity. Sebuah APK secara otomatis digunakan untuk perangkat Anda ketika Anda memilih File > Build & Run. More info
Lihat di Glossary atau AAB. Untuk informasi tentang cara menimpa isi dari Android App Manifest, lihat Mempertahankan Android App Manifest.

Permissions

Unity secara otomatis menambahkan izin yang diperlukan untuk manifestasi berdasarkan API Pengaturan Pemain Android dan Unity yang panggilan aplikasi Anda dari C# scriptsSepotong kode yang memungkinkan Anda untuk membuat Komponen Anda sendiri, memicu peristiwa permainan, memodifikasi sifat komponen dari waktu ke waktu dan menanggapi input pengguna dengan cara apa pun yang Anda sukai. More info
Lihat di Glossary. Contoh:

  • Network kelas menambahkan izin INTERNET.
  • Menggunakan getaran (seperti Login Login) menambahkan VIBRATE.
  • Properti InternetReachability menambah ACCESS_NETWORK_STATE.
  • API Lokasi (seperti LocationService) menambahkan ACCESS_FINE_LOCATION
  • WebCamTexture API add CAMERA.
  • Kelas Mikrofon menambahkan RECORD_AUDIO.

Jika plug-in membutuhkan izin yang dinyatakan dalam manifestasinya, Unity secara otomatis menambahkan izin ke final Android App Manifest selama tahap gabungan Gradle. Perhatikan bahwa Unity mencakup semua API Unity yang digunakan plug-ins dalam daftar izin.

Anda dapat menggunakan Android Sistem Izin Runtime untuk meminta izin pada runtime, bukan menentukan izin di Android App Manifest.

Untuk informasi lebih lanjut tentang izin, lihat dokumentasi pengembang Android pada Aplikasi Android Manifest Izin.
