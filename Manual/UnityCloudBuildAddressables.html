Menggunakan Alamat dalam Build Unity Cloud

Unity merekomendasikan bahwa Anda bekerja yang nyaman dengan Alamat secara lokal sebelum Anda membangunnya dengan Build Cloud.

Mengkonfigurasi Target Membangun

Jika Anda mulai dari proyek Cloud Build baru, Anda harus membuat Target Build baru terlebih dahulu.

  1. Dari Dashboard Build Cloud, arahkan ke halaman build target Config untuk proyek Anda.
  2. Pada target build yang ingin Anda bangun ditujukan untuk, memperluas menu Addressable Assets Options.
  3. Klik pada Edit Addressable Asset Options.

Gunakan halaman Addressables Options untuk mengonfigurasi properti yang menentukan bagaimana proses pembuatan Alamat berperilaku. Setiap properti dijelaskan dalam lebih detail dengan tooltip.

Menjalankan build Alamat baru

Untuk menjalankan build Alamat baru:

  1. Klik Edit Addressables Asset Options.
  2. Diaktifkan Build Addressables.
  3. Simpan konfigurasi Target.
  4. Membangun Target.

Layanan Pengiriman Konten Cloud yang Dapat Diaktifkan

Jika Anda ingin mengunggah Alamat ke layanan Pengiriman Konten Cloud (CCD) dari Cloud Build:

  1. Klik Edit Addressables Asset Options.
  2. Enable Content Delivery Upload untuk mengunggah Alamat ke CCD.
  3. Pilih ember CCD yang diperlukan dalam drop-down Content Delivery Bucket.
  4. Content Delivery API Key Anda sudah harus diisi di bidangnya. Jika tidak, tambahkan.
  5. Enable Content Delivery Create Release jika Anda ingin secara otomatis membuat pelepasan CCD dari bucket yang dipilih.
  6. Simpan konfigurasi Target.
  7. Membangun Target.

Lihat informasi lebih lanjut tentang menggunakan Alamat dengan CCD.

Menjalankan build pembaruan konten

Pembaruan konten membangun memperbarui pemain yang sebelumnya dibangun dengan konten yang dapat ditujukan baru.

Untuk memperbarui pemain yang ada, memperbarui build memerlukan file Konten State. Ini menghubungkan konten dari build Update ke pemain yang ada. File Konten Negara dihasilkan ketika Anda melakukan new Addressables build.

Untuk memperbarui pemain yang sebelumnya dibangun dengan aset yang dapat dijawab baru:

  1. Klik Edit Addressables Asset Options.
  2. Diaktifkan Build Addressables dan Content Update Build.
  3. Aktifkan Content Only Build (kecuali Anda ingin menghasilkan Pemain lain).
  4. Mengatur Content State Location Method untuk menentukan bagaimana Cloud Build mendapatkan file Konten State. Login Cloud Build bisa mendapatkan file ini dengan cara-cara berikut.
    Cloud Build can get this file in the following ways.
    • Dari kendali sumber; dalam hal mana Anda harus menentukan jalan.
    • Dari target build yang ada yang sebelumnya berhasil membangun Alamat; dalam hal mana Anda harus menentukan target.

Cloud Build dapat secara otomatis menggunakan file Konten Negara terbaru yang diproduksi oleh Target Build yang dipilih.

Mulai membangun

Ketika target build dikonfigurasi, untuk memulai build Alamat baru:

  1. Navigasi ke halaman Build History dari dashboard.
  2. Bangun target yang telah dikonfigurasi di Alamat.

Ketika build baru berhasil selesai, menu Lainnya (Perusahaan) untuk membangun menampilkan opsi Download Addressable Assets.

Menyalin konten Alamat dari Cloud Build ke penyedia hosting Anda

Setelah Anda menyelesaikan build Alamat, Anda dapat menyalin konten Alamat dari Cloud Build ke penyedia hosting Anda. Anda dapat melakukan ini secara manual, atau dengan menggunakan script pasca-build.

Menyalin konten yang dapat diakses secara manual

Setelah Anda menyelesaikan build Alamat, pilih Download Addressables Assets dari menu Lainnya (Periksa). Ketika Anda telah mengunduh konten, Anda dapat meng-uploadnya ke penyedia hosting Anda seperti yang biasanya akan ketika membangun lokal.

Menyalin konten yang dapat diakses dengan skrip pasca-build

Cloud Build mendukung menjalankan skrip shell kustom sebelum atau sesudah build. Anda dapat membuat skrip membangun posting untuk secara otomatis meng-upload Anda Isi alamat ketika membangun berhasil menyelesaikan.

Untuk mengaktifkan script pasca-build:

  1. Kunjungi halaman Cloud Build Config.
  2. Di Target Build Anda, pilih Edit Advanced Options.
  3. Dalam bidang Post-Build Script Path, masukkan jalan ke skrip Anda.

Kiat berikut mungkin berguna untuk mempertimbangkan ketika menulis skrip Anda:

  • Konten yang dapat dihubungi untuk membangun Anda terletak di $WORKSPACE/.build/last/&lt;BUILD_TARGET_ID&gt;/extra_data/addrs/

  • Anda dapat mengatur variabel lingkungan untuk kunci rahasia di halaman Build Target Config page
