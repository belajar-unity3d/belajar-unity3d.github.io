Menginstal dari URL Git

Manajer Paket dapat memuat paket dari repositori Git pada server jarak jauh.

Untuk informasi lebih lanjut tentang menggunakan Git dependenciesManajer Paket mengambil Ketergantungan git dari repositori Git langsung daripada dari registry paket. Ketergantungan Git menggunakan referensi URL Git bukan versi, dan tidak ada jaminan tentang kualitas paket, stabilitas, validitas, atau bahkan apakah versi yang tercantum dalam berkas package.json menghormati aturan Versi Semantic sehubungan dengan rilis resmi paket ini. More info
Lihat di Glossary dengan Unity Package Manager, lihat dokumentasi menggunakan Git URLs di project manifestSetiap proyek Unity memiliki manifestasi proyek, yang bertindak sebagai titik masuk untuk Manajer Paket. File ini harus tersedia di direktori <project>/Packages. Manajer Paket menggunakannya untuk mengkonfigurasi banyak hal, termasuk daftar dependensi untuk proyek itu, serta repositori paket untuk permintaan. More info
Lihat di Glossary Anda.

Untuk memuat paket dari URL Git:

  1. Klik tombol add di bilah status.

  2. Pilihan untuk menambahkan paket muncul.

  3. Pilih Add package from git URL dari menu tambahan. Kotak teks dan tombol Add muncul.

  4. Masukkan URL Git yang valid di kotak teks dan klik Add.

    Unity mendukung beberapa Protokol Git dan beberapa sintaks kustom atau diperpanjang. Untuk informasi lebih lanjut tentang cara membangun URL Git yang valid, lihat URL Git dan sintaks yang diperpanjang.

Jika Unity dapat menginstal paket berhasil, paket sekarang muncul dalam label Daftar paket dengan label . Jika Unity tidak dapat menginstal paket, Konsol Unity menampilkan pesan kesalahan, seperti:

  • Tidak ada executable ‘git’ ditemukan
  • git-lfs: perintah tidak ditemukan
  • Repositori tidak ditemukan
  • Tidak bisa membaca Nama Pengguna: terminal prompt dinonaktifkan

Klik pada tautan pesan kesalahan untuk mendapatkan beberapa bantuan di halaman Troubleshooting.

Tip: Jika Anda ingin memeriksa pembaruan dan memperbarui Anda Ketergantungan git ke versi terbaru dari repositori, klik Update. Anda juga dapat menggunakan item menu Tambahkan paket dari git URL untuk memperbarui ketergantungan Git Anda. Untuk informasi mengenai ketergantungan Git, lihat Ketergantungan Git yang dikunci.
