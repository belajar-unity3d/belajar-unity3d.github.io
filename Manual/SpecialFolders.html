Nama folder khusus

Anda biasanya dapat memilih nama yang Anda sukai untuk folder yang Anda buat untuk mengatur proyek Unity Anda. Namun, ada nama folder yang cadangan Unity untuk tujuan khusus. Misalnya, Anda harus menempatkan Editor scriptsSepotong kode yang memungkinkan Anda untuk membuat Komponen Anda sendiri, memicu peristiwa permainan, memodifikasi sifat komponen dari waktu ke waktu dan menanggapi input pengguna dengan cara apa pun yang Anda sukai. More info
Lihat di Glossary dalam folder yang disebut Editor untuk mereka bekerja dengan benar.

Halaman ini berisi daftar lengkap nama folder khusus yang digunakan oleh Unity.

Assets

Folder AssetsSetiap media atau data yang dapat digunakan dalam permainan atau proyek Anda. Aset mungkin berasal dari file yang dibuat di luar Unity, seperti model 3D, file audio atau gambar. Anda juga dapat membuat beberapa jenis aset di Unity, seperti Pengontrol Animator, Audio Mixer atau Tekstur Render. More info
Lihat di Glossary adalah folder utama yang berisi Aset yang digunakan oleh proyek Unity. Konten Project windowJendela yang menunjukkan isi folder Assets (Project tab) More info
Lihat di Glossary dalam Editor sesuai dengan isi folder Aset. Sebagian besar fungsi API mengasumsikan bahwa semuanya terletak di folder Aset, dan tidak memerlukannya untuk disebutkan secara eksplisit. Namun, beberapa fungsi perlu memiliki folder Aset termasuk sebagai bagian dari nama jalan (misalnya, fungsi tertentu dalam kelas AssetDatabase).

Editor

Skrip editor menambahkan fungsi ke Unity selama perkembangan, tetapi tidak tersedia dalam builds pada runtime. Script di folder Editor dijalankan sebagai skrip Editor, bukan skrip runtime.

Anda dapat memiliki beberapa folder Editor yang ditempatkan di mana saja di dalam folder Aset. Tempatkan skrip Editor Anda di dalam folder Editor atau subfolder di dalamnya.

Lokasi yang tepat dari folder Editor mempengaruhi waktu di mana skripnya dikompilasi relatif terhadap skrip lain. Lihat dokumentasi tentang Folder Khusus dan Urutan Script untuk deskripsi penuh.

Gunakan fungsi EditorGUIUtility.Load dalam skrip Editor untuk memuat Aset dari folder Sumber Daya dalam folder Editor. Aset ini hanya dimuat melalui skrip Editor, dan dilucuti dari build.

Sitemap Unity tidak memungkinkan komponen yang berasal dari MonoBehaviour untuk ditugaskan ke Note: jika skrip berada di folder Editor.GameObjectsObjek mendasar dalam adegan Unity, yang dapat mewakili karakter, props, pemandangan, kamera, waypoints, dan banyak lagi. Fungsi GameObject didefinisikan oleh Komponen yang melekat padanya. More info
Lihat di Glossary if the scripts are in the Editor folder.

Editor Default Resources

Skrip editor dapat menggunakan file Asset yang dimuat pada-demand menggunakan fungsi EditorGUIUtility.Load. Fungsi ini terlihat untuk file Asset dalam folder yang disebut Editor Default Resources.

Anda hanya dapat memiliki satu folder Sumber Daya Default Editor dan Anda harus menempatkannya di akar Proyek, langsung dalam folder Aset. Tempatkan file Aset yang diperlukan di folder Sumber Daya Default Editor ini, atau subfolder di dalamnya. Selalu termasuk jalur subfolder di jalan yang dilewatkan ke fungsi EditorGUIUtility.Load jika file Aset Anda berada di subfolder.

Gizmos

Gizmos memungkinkan Anda untuk menambahkan grafik ke SceneAdegan berisi lingkungan dan menu permainan Anda. Pikirkan setiap file Adegan unik sebagai tingkat yang unik. Di setiap Adegan, Anda menempatkan lingkungan, hambatan, dan dekorasi, pada dasarnya merancang dan membangun permainan Anda dalam potongan-potongan. More info
Lihat di Glossary Lihat untuk membantu memvisualisasikan rincian desain yang tidak terlihat. Fungsi Gizmos.DrawIcon menempatkan ikon di Adegan untuk bertindak sebagai penanda untuk objek atau posisi khusus. Anda harus menempatkan file gambar yang digunakan untuk menggambar ikon ini dalam folder yang disebut GizmosOverlay grafis yang terkait dengan GameObject di Adegani, dan ditampilkan di Tampilan Adegan. Alat adegan internal seperti alat bergerak adalah Gizmos, dan Anda dapat membuat Gizmos kustom menggunakan tekstur atau scripting. Beberapa Gizmos hanya ditarik ketika GameObject dipilih, sementara Gizmos lainnya ditarik oleh Editor terlepas dari GameObjects mana yang dipilih. More info
Lihat di Glossary untuk fungsi DrawIcon untuk menemukannya.

Anda hanya dapat memiliki satu folder Gizmos dan harus ditempatkan dalam akar Proyek, langsung dalam folder Aset. Tempatkan file Aset yang diperlukan di folder Gizmos ini atau subfolder di dalamnya. Selalu termasuk jalur subfolder di jalan yang dilewatkan ke fungsi Gizmos.DrawIcon jika Anda File aset dalam subfolder.

Resources

Anda dapat memuat Aset on-demand dari skrip bukan membuat contoh Aset di Adegan untuk digunakan dalam gameplay. Anda melakukan ini dengan menempatkan Aset di folder yang disebut Resources. Beban Aset ini dengan menggunakan fungsi Sitemap Login.

Anda dapat memiliki beberapa folder Sumber Daya yang ditempatkan di mana saja di dalam folder Aset. Tempatkan file Aset yang diperlukan dalam folder Sumber Daya atau subfolder di dalamnya. Selalu termasuk jalur subfolder di jalan yang dilewatkan ke fungsi Sitemap Login jika file Aset Anda berada di subfolder.

Note: Jika folder Sumber Daya adalah subfolder Editor, Aset di dalamnya dimuat dari skrip Editor, tetapi dihapus dari build.

Aset Standar

Ketika Anda Standar Paket aset, Unity menempatkan aset dalam folder yang disebut Standard Assets. Serta mengandung aset, folder ini juga memiliki efek pada urutan kompilasi skrip. Untuk informasi lebih lanjut, lihat halaman di Folder Khusus dan Urutan Script.

Anda hanya dapat memiliki satu folder Standard Assets dan Anda harus meninggalkannya di akar proyek, langsung dalam folder Assets. Tempatkan file aset yang Anda butuhkan di dalam folder <project-root>/Assets/Standard Assets atau salah satu subfolder.

StreamingAssets

Anda mungkin ingin Aset tersedia sebagai file terpisah dalam format aslinya (meskipun itu lebih umum untuk langsung menggabungkan Aset menjadi build). Misalnya, Anda perlu mengakses file video dari sistem file untuk bermain video di IOS menggunakan .

Untuk menyertakan Aset Streaming, lakukan berikut:

  1. Tempatkan file di folder StreamingAssets.
  2. File tetap tidak berubah ketika disalin ke mesin target, di mana tersedia dari folder tertentu.

Lihat halaman tentang Aset Streaming untuk rincian lebih lanjut.

Anda hanya dapat memiliki satu folder StreamingAssets dan harus ditempatkan di akar Proyek, langsung dalam folder Aset. Tempatkan file Aset di folder StreamingAssets atau subfolder. Selalu termasuk jalur subfolder di jalur yang digunakan untuk merujuk aset streaming jika file Aset Anda berada di subfolder.

Aset Tersembunyi

Selama proses impor, Unity mengabaikan file dan folder berikut di folder Assets (atau sub-folder di dalamnya):

  • folder tersembunyi.
  • File dan folder yang dimulai dengan '.'.
  • File dan folder yang berakhir dengan '~'.
  • File dan folder bernama cvs.
  • File dengan ekstensi .tmp.

Ini mencegah mengimpor file khusus dan sementara yang dibuat oleh sistem operasi atau aplikasi lainnya.
