Proyek 2D atau 3D

Unity sama cocok untuk membuat game 2D dan 3D. Ketika Anda membuat proyek baru di Unity, Anda memiliki pilihan untuk memulai dalam mode 2D atau 3D. Anda mungkin sudah tahu apa yang ingin Anda bangun, tetapi ada beberapa titik halus yang dapat mempengaruhi mode mana yang Anda pilih.

Pilihan antara awal dalam mode 2D atau 3D menentukan beberapa pengaturan untuk Editor Unity, seperti apakah gambar diimpor sebagai tekstur atau spritesObjek grafis 2D. Jika Anda digunakan untuk bekerja di 3D, Sprites pada dasarnya hanya tekstur standar tetapi ada teknik khusus untuk menggabungkan dan mengelola tekstur sprite untuk efisiensi dan kenyamanan selama perkembangan. Penuh 3D

Game 3D biasanya menggunakan geometri tiga dimensi, dengan Bahan dan Tekstur yang diberikan pada permukaan GameObjectsObjek mendasar dalam adegan Unity, yang dapat mewakili karakter, props, pemandangan, kamera, waypoints, dan banyak lagi. Fungsi GameObject didefinisikan oleh Komponen yang melekat padanya. More info
Lihat di Glossary untuk membuat mereka muncul sebagai lingkungan padat, karakter dan benda-benda yang membuat dunia permainan Anda. Di setiap Adegan, Anda menempatkan lingkungan, hambatan, dan dekorasi, pada dasarnya merancang dan membangun permainan Anda dalam potongan-potongan. More info
Lihat di Glossary bebas, dengan cahaya dan bayangan yang melemparkan di seluruh dunia dengan cara yang realistis. Game 3D biasanya render adegan menggunakan perspektif, sehingga benda muncul lebih besar di layar karena mereka lebih dekat ke kamera. Untuk semua game yang sesuai dengan deskripsi ini, mulai dalam mode 3D.

Orthographic 3D

Kadang-kadang permainan menggunakan geometri 3D, tetapi gunakan kamera ortopografi bukan perspektif. Ini adalah teknik umum yang digunakan dalam permainan yang memberi Anda pandangan mata unggas dari tindakan, dan kadang-kadang disebut “2.5D”. Jika Anda membuat permainan seperti ini, Anda juga harus menggunakan Editor dalam mode 3D, karena meskipun tidak ada perspective, Anda masih akan bekerja dengan model 3D dan Aset. Penuh 2D

Banyak game 2D menggunakan grafis datar, kadang-kadang disebut kecambah, yang tidak memiliki geometri tiga dimensi sama sekali. Mereka ditarik ke layar sebagai gambar datar, dan kamera game tidak memiliki perspektif. Untuk jenis permainan ini, Anda harus memulai editor dalam mode 2D.

Gameplay 2D dengan grafis 3D

Beberapa game 2D menggunakan geometri 3D untuk lingkungan dan karakter, tetapi membatasi gameplay untuk dua dimensi. Sebagai contoh, kamera dapat menunjukkan pandangan bergeser sisi, dan pemain hanya dapat bergerak dalam dua dimensi, tetapi permainan itu sendiri masih menggunakan model 3D untuk hambatan dan perspektif 3D untuk kamera. Untuk game ini, efek 3D dapat melayani gaya daripada tujuan fungsional. Jenis permainan ini adalah also kadang-kadang disebut sebagai “2.5D”. Meskipun gameplay adalah 2D, Anda sebagian besar memanipulasi model 3D untuk membangun permainan, sehingga Anda harus memulai editor dalam mode 3D.

2D gameplay dan grafis dengan kamera perspektif

Ini adalah gaya populer lain dari permainan 2D, menggunakan grafis 2D tetapi dengan kamera perspektif untuk mendapatkan efek penggulir parallax. Ini adalah adegan bergaya "kartu", di mana semua grafis datar, tetapi diatur pada jarak yang berbeda dari kamera. Ini kemungkinan besar bahwa mode 2D akan sesuai dengan perkembangan Anda dalam kasus ini. Namun, Anda harus mengubah mode proyeksi Kamera Anda menjadi Perspective dan mode Tampilan adegan hingga 3D.
