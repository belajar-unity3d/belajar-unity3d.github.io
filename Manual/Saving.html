Menyimpan pekerjaan Anda

Klasifikasi yang paling menghemat informasi menjadi Perubahan adegan dan Perubahan luas proyek.

  • Untuk menyimpan semua perubahan adegan dan perubahan luas proyek, pergi ke File > Save (atau Save as). Ini adalah cara tercepat untuk menyimpan semuanya sekaligus.
  • Untuk menyimpan perubahan luas proyek, tetapi tidak perubahan adegan, pergi ke File > Save Project.

Note: Pengecualian untuk perilaku ini terjadi ketika Anda mengedit di Mode Prefab. Dalam kasus ini, File > Save hanya menghemat perubahan pada Prefab terbuka. Ini tidak menyimpan perubahan adegan atau perubahan luas proyek.

Unity juga menyimpan beberapa informasi segera saat Anda bekerja di Editor. Lihat Hemat Segera, di bawah ini, untuk rincian lebih lanjut.

Adegan perubahan

Perubahan adegan termasuk modifikasi untuk GameObjectsObjek mendasar dalam adegan Unity, yang dapat mewakili karakter, props, pemandangan, kamera, waypoints, dan banyak lagi. Fungsi GameObject didefinisikan oleh Komponen yang melekat padanya. More info
Lihat di Glossary di SceneAdegan berisi lingkungan dan menu permainan Anda. Pikirkan setiap file Adegan unik sebagai tingkat yang unik. Di setiap Adegan, Anda menempatkan lingkungan, hambatan, dan dekorasi, pada dasarnya merancang dan membangun permainan Anda dalam potongan-potongan. More info
Lihat di Glossary, seperti ketika Anda:

  • Menambahkan, memindahkan atau menghapus GameObject
  • Mengubah parameter GameObject di Inspektur

Perubahan luas proyek

Perubahan luas proyek dalam Unity berlaku untuk seluruh proyek Anda, daripada adegan tertentu. Jika Anda ingin menyimpan pengaturan luas proyek tanpa menyimpan perubahan adegan, pergi ke File > Save Project. Anda mungkin menemukan ini berguna jika, misalnya, Anda membuat adegan sementara untuk menguji beberapa perubahan.

Perubahan luas proyek meliputi:

  • Project SettingsPengumpulan yang luas dari pengaturan yang memungkinkan Anda untuk mengkonfigurasi bagaimana Fisika, Audio, Jaringan, Grafik, Input dan banyak bidang lain dari jangkauan proyek Anda. More info
    Lihat di Glossary    Library ketika Proyek disimpan. Ini menyimpan pengaturan dalam file berikut:
    • Input: InputManager.asset
    • Tags And Layers: TagManager.asset
    • Audio: AudioManager.asset
    • Time: TimeManager.asset
    • Player: ProjectSettings.asset
    • Physics: DynamicsManager.asset
    • Physics 2D: Fisika2DSettings.asset
    • Quality: QualitySettings.asset
    • Graphics: Grafik
    • Network: WordPress.org
    • Editor: WordPress.org
  • Build Settings: Unity menyimpan perubahan pada Pengaturan Build di folder Library sebagai WordPress.org.
  • Changed Assets: Selama simpanan yang menyimpan pengaturan di seluruh proyek, Unity menyimpan aset perubahan yang belum disadari. Hemat Segera

Ada beberapa perubahan yang Unity segera menyimpan untuk disk ketika mereka terjadi. Ini termasuk berikut:

  • New Assets: Unity secara otomatis menyimpan Aset baru pada penciptaan, tetapi tidak perubahan berikutnya pada Aset tersebut.
  • Asset Import Settings: Pengaturan Impor untuk sebagian besar jenis Aset memerlukan bahwa Anda menekan tombol “Apply” untuk perubahan untuk mengambil efek. Unity menyimpan perubahan ini segera setelah Anda memilih Berlaku.
  • Baked data: Ada beberapa jenis data yang Anda "bake" ke dalam Proyek Anda. Unity menyimpan data ini secara otomatis ketika setiap selesai bake. Ini termasuk:
    • Foto di Baked Lighting
    • Data navigasi panggang
    • Foto di Baked occlusion cullingA yang menonaktifkan rendering objek ketika mereka tidak saat ini dilihat oleh kamera karena mereka dikabur (disebutkan) oleh benda lain. More info
      Lihat di Glossary       data
  • Script execution order changes: Setelah Anda memilih Apply, Unity segera menyimpan data ini ke setiap file .meta script.
