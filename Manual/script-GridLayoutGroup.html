Grid Layout Group

Komponen Grid Layout Group menempatkan elemen tata letak anaknya di grid.

Properties

Property: Function:
Padding Padding di dalam tepi kelompok tata letak.
Cell Size Ukuran untuk digunakan untuk setiap elemen tata letak dalam kelompok.
Spacing Jarak antara elemen tata letak.
Start Corner Sudut di mana elemen pertama terletak.
Start Axis Sumbu utama mana untuk menempatkan unsur-unsur di sepanjang. Horizontal akan mengisi seluruh baris sebelum baris baru dimulai. Vertikal akan mengisi seluruh kolom sebelum kolom baru dimulai.
Child Alignment Penyelarasan untuk digunakan untuk elemen tata letak jika mereka tidak mengisi semua ruang yang tersedia.
Constraint Melanjutkan grid ke jumlah baris atau kolom tetap untuk membantu sistem tata letak otomatis.

Description

Tidak seperti kelompok tata letak lainnya, Grup Tata Kelola Grid mengabaikan sifat ukuran minimum, disukai, dan fleksibel dari elemen tata letak yang terkandung dan bukan menetapkan ukuran tetap bagi semua dari mereka yang didefinisikan dengan properti Ukuran Sel Grup Tata Letak Grid itu sendiri.

Kelompok Tata Letak Grid dan tata letak otomatis

Ada pertimbangan khusus untuk menyadari ketika menggunakan Grid Layout Group sebagai bagian dari pengaturan tata letak otomatis, seperti menggunakannya dengan Ukuran Konten Fitter.

Sistem tata letak otomatis menghitung ukuran horisontal dan vertikal secara independen. Hal ini dapat terjadi pada odds dengan Grid Layout Group, di mana jumlah baris tergantung pada jumlah kolom dan sebaliknya.

Untuk setiap jumlah sel yang diberikan, ada kombinasi yang berbeda dari jumlah baris dan jumlah kolom yang dapat membuat grid sesuai kontennya. Untuk membantu sistem tata letak, Anda dapat menentukan bahwa Anda mengutip tabel untuk memiliki jumlah kolom atau baris tetap dengan menggunakan properti Constraint.

Berikut ini adalah cara yang disarankan untuk menggunakan Sistem Tata Letak dengan Ukuran Konten Fitter:

Lebar fleksibel dan tinggi tetap

Untuk mengatur grid dengan lebar fleksibel dan tinggi tetap, di mana grid memperluas secara horizontal sebagai lebih banyak elemen ditambahkan, Anda dapat mengatur sifat ini sebagai berikut:

  • Kontraint Kelompok Tata Letak Grid: Hitungan Baris Tetap
  • Ukuran Konten Fitter Horizontal Fit: Ukuran yang disukai
  • Ukuran Konten Fitter Vertical Fit: Ukuran yang disukai atau Tidak Terkendali

Jika unconstrained Vertical Fit digunakan, itu hingga Anda untuk memberikan grid tinggi yang cukup besar untuk menyesuaikan jumlah baris tertentu sel.

Lebar tetap dan tinggi yang fleksibel

Untuk mengatur grid dengan lebar tetap dan tinggi yang fleksibel, di mana grid memperluas secara vertikal sebagai lebih elemen ditambahkan, Anda dapat mengatur sifat ini sebagai berikut:

  • Kontraint Kelompok Tata Letak Grid: Count Kolom Tetap
  • Ukuran Konten Fitter Horizontal Fit: Ukuran yang disukai atau Tidak Terkendali
  • Ukuran Konten Fitter Vertikal Fit: Ukuran yang disukai

Jika unconstrained Horizontal Fit digunakan, itu hingga Anda untuk memberikan grid lebar yang cukup besar untuk menyesuaikan jumlah kolom sel yang ditentukan.

Lebar fleksibel dan tinggi

Jika Anda ingin grid dengan lebar fleksibel dan tinggi Anda dapat melakukannya, tetapi Anda tidak akan memiliki kontrol atas jumlah baris dan kolom tertentu. Grid akan mencoba membuat baris dan jumlah kolom sekitar sama. Anda dapat mengatur properti ini sebagai berikut:

  • Kontraint Kelompok Tata Letak Grid: Fleksibel
  • Ukuran Konten Fitter Horizontal Fit: Ukuran yang disukai
  • Ukuran Konten Fitter Vertikal Fit: Ukuran yang disukai
