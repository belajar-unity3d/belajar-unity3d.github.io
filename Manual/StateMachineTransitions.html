Transisi Mesin Negara

Transisi Mesin Negara ada untuk membantu Anda menyederhanakan Mesin Negara besar atau kompleks. Mereka memungkinkan Anda untuk memiliki tingkat abstraksi yang lebih tinggi atas logika state machineSatu set negara di Animator Controller yang karakter atau animasi GameObject dapat masuk, bersama dengan satu set transisi antara negara-negara tersebut dan variabel untuk mengingat keadaan saat ini. Negara-negara yang tersedia akan tergantung pada jenis gameplay, tetapi keadaan khas termasuk hal-hal seperti idling, berjalan, berjalan dan melompat. More info

Lihat di Glossary memiliki node Masuk dan Keluar. Ini digunakan selama Transisi Mesin Negara.

Node entri digunakan ketika transisi ke mesin negara. Node entri akan dievaluasi dan akan bercabang ke negara tujuan sesuai dengan ketentuan yang ditetapkan. Dengan cara ini node entri dapat mengontrol mana keadaan mesin dimulai, dengan mengevaluasi keadaan parameter Anda ketika mesin negara dimulai.

Karena mesin negara selalu memiliki state default, akan selalu ada percabangan transisi default dari node masuk ke state default.

Anda kemudian dapat menambahkan transisi tambahan dari node Masuk ke negara lain, untuk mengontrol apakah mesin negara harus dimulai dalam keadaan yang berbeda.

Node Keluar digunakan untuk menunjukkan bahwa mesin negara harus keluar.

Setiap sub-state dalam mesin state dianggap sebagai mesin state terpisah dan lengkap, sehingga dengan menggunakan node masuk dan keluar ini, Anda dapat mengontrol aliran dari mesin state tingkat atas ke mesin sub-state lebih elegan.

Hal ini dimungkinkan untuk mencampur transisi mesin negara dengan transtisisi state biasa, sehingga dimungkinkan untuk transisi dari state ke statemachine, dari satu statemachine langsung ke mesin state lainnya.
