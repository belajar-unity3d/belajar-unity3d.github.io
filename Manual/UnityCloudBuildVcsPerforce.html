Menggunakan Dashboard Pengembang Unity untuk mengkonfigurasi Build Cloud untuk Perforce

Membangun Unity Cloud mendukung Proyek yang disimpan dalam repositories Perforce.

Penutup topik ini:

  • Mengkonfigurasi Perforce di Dasbor
  • Menyiapkan platform membangun target

Mengkonfigurasi Perforce di Dasbor

Untuk mengkonfigurasi Cloud Membangun Proyek Anda dari Repositori Perforce:

  1. Masuk ke .

  2. Pada halaman ProjectsDalam Unity, Anda menggunakan proyek untuk merancang dan mengembangkan permainan. Proyek menyimpan semua file yang terkait dengan permainan, seperti file aset dan Adegan. More info
    Lihat di Glossary    , pilih Proyek Anda.

  3. Di jendela dashboard Overview, pada ubin Cloud BuildLayanan integrasi berkelanjutan untuk proyek Unity yang mengotomatiskan proses pembuatan build pada server Unity. More info
    Lihat di Glossary    , klik OPEN CLOUD BUILD DASHBOARD.

  4. Di jendela Build History, klik Set up Cloud Build.

  5. Pilih platform mana untuk membangun permainan Anda.

  6. Di jendela Source Control, klik tab MANUAL.

  7. Dalam bidang SCM URL, masukkan URL server Perforce Anda di kedua format berikut:

    • Non-SSL: host: pelabuhan
    • WordPress.org

  8. Untuk menghubungkan ke repositori Anda, Anda harus menentukan URL ke server Perforce Anda. Untuk informasi tentang format URL, lihat bagian URL syntax di bawah ini.

  9. Dari menu drop-down SCM Types, pilih PerforceSistem kontrol versi untuk manajemen perubahan file. More info
    Lihat di Glossary    .

  10. Klik tombol NEXT: ACCESS. Ketika Unity Cloud Build terhubung ke situs hosting, secara otomatis mendeteksi apakah repositori Anda adalah publik atau pribadi. Jika repositori Anda adalah publik, Cloud Build secara otomatis terhubung kenya dan Anda dapat melewatkan ke Menyiapkan platform membangun target.

  11. Pada jendela Grant us access to your source control, masukkan Username dan Password untuk repositori Perforce Anda.

  12. Klik NEXT: TARGET SETUP.

URL

Untuk menghubungkan ke repositori Anda, Anda harus menentukan URL ke server Perforce Anda. Anda dapat menentukan URL untuk menggunakan protokol https:// atau SSL.

Berikut ini adalah contoh URL yang valid:

  • url https://127.0.0.1:1667
  • ssl:127.0.0.1:167

Menyiapkan platform membangun target

Di dasbor, di jendela NEW BUILD TARGET: BASIC INFO:

  1. Dalam bidang Target Label, masukkan nama untuk membangun.
  2. Dari menu drop-down Client Workspace, pilih ruang kerja dari mana untuk membangun Proyek Anda.
  3. Jika folder akar repositori Anda tidak mengandung Assets dan Project settingsPengumpulan yang luas dari pengaturan yang memungkinkan Anda untuk mengkonfigurasi bagaimana Fisika, Audio, Jaringan, Grafik, Input dan banyak bidang lain dari jangkauan proyek Anda. More info
    Lihat di Glossary     Anda, di bidang Project Subfolder, masukkan URL folder yang berisi Aset dan pengaturan Proyek Anda.
  4. Dari menu drop-down Unity Version, pilih versi Unity yang harus membangun Proyek.
  5. Jika Anda tidak ingin Proyek untuk secara otomatis membangun setiap repositori Anda diperbarui, klik toggle Auto-build untuk menonaktifkan fitur ini.

Jika Anda membangun platform iOS atau Android, langkah berikutnya Anda adalah memasukkan kredensial untuk membangun. Klik Next: Credentials. Untuk semua platform lain, klik Next: Build untuk menyelesaikan konfigurasi dan memulai build awal.

Android

Di jendela Android SIGNING CREDS, memberikan informasi berikut:

  • Sebuah Bundle ID untuk mengidentifikasi aplikasi Anda secara unik di perangkat dan di Google Play Store.
  • Masukkan kredensial Android keystoreSistem Android yang memungkinkan Anda menyimpan entri kunci kriptografi untuk keamanan perangkat yang ditingkatkan. More info
    Lihat di Glossary     atau pilih Auto Generated Debug Keystore untuk menggunakan keystore pengembangan. Untuk informasi lebih lanjut tentang keystore Android, lihat Sistem Keystore Android.

  • A Bundle ID untuk mengidentifikasi aplikasi Anda secara unik di perangkat.
  • Versi Xcode yang untuk membangun aplikasi.
  • Masukkan kredensial iOS Anda. Untuk informasi lebih lanjut tentang kredensial iOS, lihat API.
