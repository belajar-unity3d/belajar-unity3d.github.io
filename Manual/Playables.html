API documentation

API Playables menyediakan cara untuk membuat alat, efek atau mekanisme gameplay lainnya dengan mengatur dan mengevaluasi sumber data dalam struktur seperti pohon yang dikenal sebagai PlayableGraph. PlayableGraph memungkinkan Anda untuk mencampur, mencampur, dan memodifikasi beberapa sumber data, dan memainkannya melalui output tunggal.

API Playables mendukung animasi, audio dan scriptsSepotong kode yang memungkinkan Anda untuk membuat Komponen Anda sendiri, memicu peristiwa permainan, memodifikasi sifat komponen dari waktu ke waktu dan menanggapi input pengguna dengan cara apa pun yang Anda sukai. More info
Lihat di Glossary. API Playables juga menyediakan kapasitas untuk berinteraksi dengan sistem audio sistem animasi dan melalui scripting.

Meskipun API Playables saat ini terbatas pada animasi, audio, dan skrip, itu adalah API generik yang akhirnya akan digunakan oleh video dan sistem lainnya.

Playable vs Animasi

Sistem animasi sudah memiliki alat pengeditan grafik, sistem state machineSatu set negara di Animator Controller yang karakter atau animasi GameObject dapat masuk, bersama dengan satu set transisi antara negara-negara tersebut dan variabel untuk mengingat keadaan saat ini. Negara-negara yang tersedia akan tergantung pada jenis gameplay, tetapi keadaan khas termasuk hal-hal seperti idling, berjalan, berjalan dan melompat. More info
Lihat di Glossary yang dibatasi untuk bermain animasi. API Playables dirancang untuk lebih fleksibel dan mendukung sistem lain. API Playables juga memungkinkan pembuatan grafik tidak mungkin dengan mesin negara. Grafik ini mewakili aliran data, menunjukkan apa yang masing-masing node menghasilkan dan mengkonsumsi. Selain itu, grafik tunggal tidak terbatas pada sistem tunggal. Grafik tunggal dapat berisi node untuk animasi, audio, dan skrip.

Keuntungan menggunakan API Playables

  • API Playables memungkinkan untuk pencampuran animasi dinamis. Ini berarti bahwa objek di scenesAdegan berisi lingkungan dan menu permainan Anda. Pikirkan setiap file Adegan unik sebagai tingkat yang unik. Di setiap Adegan, Anda menempatkan lingkungan, hambatan, dan dekorasi, pada dasarnya merancang dan membangun permainan Anda dalam potongan-potongan. More info
    Lihat di Glossary     dapat memberikan animasi mereka sendiri. Misalnya, animasi untuk senjata, dada, dan perangkap dapat secara dinamis ditambahkan ke PlayableGraph dan digunakan untuk durasi tertentu.

  • API Playables memungkinkan Anda untuk dengan mudah memainkan animasi tunggal tanpa overhead yang terlibat dalam menciptakan dan mengelola aset AnimatorController.

  • API Playables memungkinkan pengguna untuk secara dinamis membuat grafik campuran dan mengontrol berat campuran langsung bingkai dengan bingkai.

  • Login Grafik dapat dibuat pada runtime, menambahkan node yang dapat dimainkan sesuai kebutuhan, berdasarkan kondisi. Alih-alih memiliki grafik “satu-size-fit-all” yang besar di mana node diaktifkan dan dinonaktifkan, PlayableGraph dapat disesuaikan dengan persyaratan situasi saat ini.
