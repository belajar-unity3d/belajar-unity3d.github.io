Data reset

Unity AnalyticsUnity Analytics
Lihat di Glossary memungkinkan Anda untuk menghapus semua data secara permanen dari dashboard Analytics Anda dengan melakukan reset data. Tindakan ini cannot be undone, jadi baca dokumentasi ini dan pesan konfirmasi di dasbor Analytics dengan cermat sebelum melakukan tindakan ini.

Halaman ini menjelaskan ketika reset data mungkin berguna, dan dampak apa yang dimiliki pada data dalam proyek Analytics Anda.

Ketika melakukan reset data

skenario utama yang menyerukan untuk mengatur ulang data dalam proyek Analytics adalah menghapus data dari dasbor Anda segera sebelum meluncurkan (atau melepaskan kembali) permainan Anda. Ini memastikan bahwa data yang Anda kumpulkan setelah ulang bersih dan tidak dicampur dengan data yang dikumpulkan sebelumnya, yang dapat mengubah analisis Anda.

skenario umum lainnya:

  • Memacu data yang dikumpulkan selama pengembangan dan pengujian sebelum meluncurkan permainan Anda.

  • Data Wiping dari versi pra-release atau beta.

  • Memutar data sebelum meluncurkan permainan Anda di toko.

  • Memancarkan data sebelum kembali produk.

Harap dicatat bahwa melakukan reset data adalah opsional - Anda mungkin tidak perlu menggunakan fitur ini.

Bagaimana dampak ini data Anda

Karena siklus pemrosesan data Unity Technologies, semua reset data terjadi pada 00:00 UTC hari berikutnya. Lihat halaman Konfigurasi untuk waktu yang tepat (dalam zona waktu Anda) data Anda disebabkan untuk diatur ulang.

Data dikumpulkan dari waktu reset terjadi, tetapi mungkin memakan waktu hingga 4 jam untuk mengisi dasbor Anda. Anda tidak dapat menentukan tanggal lain untuk mengatur ulang data Anda.

Mengatur ulang data Anda mempengaruhi elemen Analytics ini sebagai berikut:

  • Sitemap Ini secara permanen dihapus dari semua data yang terlihat, termasuk data dalam grafik.Dashboard: This is permanently wiped of all visible data, including data in charts.

  • Sitemap Ini secara permanen dihapus dari dashboard. Perhatikan bahwa peristiwa kustom yang telah Anda instrumentasi dalam proyek Anda tidak perlu divalidasi kembali.Custom event data: This is permanently wiped from the dashboard. Note that custom events you have instrumented in your project donâ€™t need to be re-validated.

  • Sitemap Semua data yang terkait dengan konfigurasi ini (yang telah Anda buat di proyek Anda) secara permanen dihapus dari dashboard. Namun, konfigurasi itu sendiri tidak dihapus. Misalnya, jika Anda membuat saluran tiga langkah, mengisinya dengan data, dan kemudian melakukan reset data, saluran tiga langkah masih ada di dasbor Anda tetapi tidak lagi diisi dengan data.Reports, Funnels, Segments: All data associated with these configurations (that you have created in your project) is permanently wiped from the dashboard. However, the configurations themselves are not deleted. For example, if you create a three-step funnel, populate it with data, and then perform a data reset, the three-step funnel is still present in your dashboard but is no longer populated with the data.

  • Sitemap Ini tidak terpengaruh dengan melakukan reset data. Anda masih dapat melakukan ekspor untuk mengakses data historis, dan semua ekspor sebelumnya yang Anda jalankan masih ada di tabel Aktivitas di halaman Ekspor Data Raw.Raw Data Export: This is not affected by performing a data reset. You can still perform an export to access historical data, and all previous exports you have run are still present in the Activity table on the Raw Data Export page.

Cara melakukan reset data

Untuk melakukan reset data, buka Dasbor Layanan Unity untuk proyek Analytics yang ingin Anda reset, pergi ke Configure, dan klik pada Reset Data. Pastikan membaca pesan konfirmasi dengan hati-hati. Ketik nama proyek Anda ke dalam kotak teks dan klik Reset. Reset data terjadi segera.

Ketika data direset, rincian berikut disediakan tentang ulang terakhir di halaman Configure:

  • Tanggal dan waktu reset terjadi.

  • Pengguna yang melakukan reset.

Sitemap Hanya Pemilik Proyek dan Manajer dapat melakukan tindakan ini. Pengguna Proyek tidak memiliki izin untuk melakukan reset.Note: Only Project Owners and Managers can perform this action. Project Users do not have permission to perform a reset.
