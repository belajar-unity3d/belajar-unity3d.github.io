Bidang Masukan

Sebuah Input Field adalah cara untuk membuat teks yang dapat diedit oleh Kontrol Teks. Seperti kontrol interaksi lain, itu bukan elemen UI(User Interface) Memungkinkan pengguna untuk berinteraksi dengan aplikasi Anda. Unity saat ini mendukung tiga sistem UI. More info
Lihat di Glossary yang terlihat sendiri dan harus dikombinasikan dengan satu atau lebih elemen UI visual untuk terlihat.

Properties

Property: Function:
Interactable Bosan yang menentukan jika Bidang Input dapat berinteraksi dengan atau tidak.
Transition TransitionsCampuran dari satu negara ke mesin negara lain, seperti transisi karakter dari berjalan ke animasi jog. Transisi menentukan berapa lama campuran antara negara harus mengambil, dan kondisi yang mengaktifkan campuran. More info
Lihat di Glossary digunakan untuk mengatur bagaimana transisi bidang masukan ketika Normal, Highlighted, Pressed atau Disabled.
Navigation Properti yang menentukan urutan kontrol. SitemapPilihan Navigasi.
TextComponent Referensi elemen TextSepotong teks non-interaktif untuk pengguna. Ini dapat digunakan untuk menyediakan pembatasan atau label untuk kontrol GUI lainnya atau untuk menampilkan instruksi atau teks lainnya. More info
Lihat di Glossary yang digunakan sebagai isi dari Bidang Masukan
Text Nilai Mulai. Teks awal yang ditempatkan di lapangan sebelum mengedit dimulai.
Character Limit Nilai jumlah karakter maksimum yang dapat dimasukkan ke dalam bidang masukan.
Content Type Tentukan jenis karakter yang menerima kolom input Anda
Standard Setiap karakter dapat dimasukkan.
Autocorrected Autocorrection menentukan apakah input melacak kata-kata yang tidak diketahui dan menyarankan kandidat pengganti yang lebih cocok untuk pengguna, mengganti teks yang diketik secara otomatis kecuali pengguna secara eksplisit menimpa aksi.
Integer Number Izinkan hanya seluruh nomor yang akan dimasukkan.
Decimal Number Memungkinkan hanya angka dan satu titik desimal yang akan dimasukkan.
Alphanumeric Memungkinkan huruf dan angka. Simbol tidak dapat dimasukkan.
Name Secara otomatis memanfaatkan surat pertama setiap kata. Perhatikan bahwa pengguna dapat menghindari aturan kapitalisasi menggunakan tombol Delete.
Email Address Memungkinkan Anda memasukkan string alfanumerik yang terdiri dari maksimum satu tanda @. periode / titik dasar tidak dapat dimasukkan ke dalam satu sama lain.
Password* Menerima karakter yang dimasukkan dengan asterisk. Memungkinkan simbol.
Pin Menerima karakter yang dimasukkan dengan asterisk. Hanya memungkinkan seluruh nomor yang akan dimasukkan.
Custom Memungkinkan Anda untuk menyesuaikan Jenis Garis, Jenis Input, Jenis Keyboard dan Validasi Karakter.
Line Type Tentukan bagaimana teks diformat di dalam bidang teks.
Single Line Hanya memungkinkan teks untuk berada di satu baris.
Multi Line Submit Memungkinkan teks untuk menggunakan beberapa baris. Hanya menggunakan garis baru bila diperlukan.
Multi Line Newline Memungkinkan teks untuk menggunakan beberapa baris. Pengguna dapat menggunakan garis baru dengan menekan kunci pengembalian.
Placeholder Ini adalah Graphic opsional untuk menunjukkan bahwa Bidang Masukan kosong teks. Perhatikan bahwa grafis ‘empty’ ini masih menampilkan bahkan ketika Bidang Masukan dipilih (yaitu; ketika ada fokus padanya). eg; "Enter teks ...".
Caret Blink Rate Tentukan tingkat blink untuk tanda yang ditempatkan di garis untuk menunjukkan penyisipan teks yang diusulkan.
Selection Color Warna latar belakang dari bagian teks yang dipilih.
Hide Mobile Input (iOS only) Sembunyikan kolom input asli yang melekat pada keyboard layar pada perangkat seluler. Perhatikan bahwa ini hanya bekerja pada perangkat iOS.

Events

Property: Function:
On Value Change Sebuah UnityEvent yang diinjak ketika konten teks dari perubahan Field Input. Acara ini dapat mengirim konten teks saat ini sebagai argumen dinamis tipe string.
End Edit Sebuah UnityEvent yang diinvokasi ketika pengguna selesai mengedit konten teks baik dengan mengirimkan atau dengan mengklik suatu tempat yang menghapus fokus dari Bidang Input. Acara ini dapat mengirim konten teks saat ini sebagai argumen dinamis tipe string.

Details

Script kolom input dapat ditambahkan ke objek kontrol teks yang ada dari menu (Component > UI > Input Field). Setelah melakukan ini, Anda juga harus menyeret objek ke properti Text kolom Input untuk mengaktifkan pengeditan.

Properti Text dari kontrol Teks sendiri akan berubah sebagai jenis pengguna dan nilai dapat diambil dari skrip setelah mengedit. Perhatikan bahwa Teks Kaya sengaja tidak didukung untuk kontrol Teks yang dapat diedit; lapangan akan menerapkan markup Teks Kaya secara langsung ketika diketik tetapi markup pada dasarnya “berhenti” dan tidak ada cara berikutnya untuk mengubah atau menghapus styling.

Hints

  • Untuk mendapatkan teks Bidang Input, gunakan properti teks pada komponen InputField itu sendiri, bukan properti teks dari komponen Teks yang menampilkan teks. Sifat teks dari komponen Teks dapat dirajut atau mungkin terdiri dari tanda kurik untuk kata sandi.
