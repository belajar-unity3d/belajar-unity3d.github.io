Mengelola dan menerbitkan permainan Anda di konsol UDP

Unggah permainan Anda ke konsol UDP

Setelah Anda membangun permainan Anda di Editor Unity, konsol UDP perlu membongkarnya dengan SDK setiap toko sebelum Anda dapat mempublikasikan permainan Anda ke toko yang didukung. Untuk mengunggah build Anda ke Konsol UDP, ikuti langkah-langkah ini. Sitemap Konsol UDP hanya menerima Note atau AAB yang memiliki implementasi UDP.APKsFormat paket Android output oleh Unity. Sebuah APK secara otomatis digunakan untuk perangkat Anda ketika Anda memilih File > Build & Run. More info
Lihat di Glossary or AABs that have a UDP implementation.

  1. Di Konsol UDP, pilih proyek Anda di My Games.
  2. Di tab Game Info, buka bagian Binary.
    • Pilih Upload APK dan unggah file APK (dan file OBB, jika ada) permainan Anda.
    • Untuk mengunggah file AAB, pilih Convert from AAB file dan unggah file.

Anda juga dapat menyebarkan permainan Anda ke konsol UDP dengan cara berikut:

  • Menggunakan API UDP
  • Unduh APK Anda ke CloudBuild, dan menyebarkannya ke UDP dari sana
  • Bangun permainan Anda dengan CloudBuild, dan menyebarkannya ke UDP dari sana

Mengungkapkan permainan Anda ke konsol UDP menggunakan API UDP

Anda hanya dapat menggunakan API untuk mengunggah file biner untuk versi draft dari permainan Anda. Jika tidak ada versi draft dari permainan Anda, UDP menciptakan satu berdasarkan versi terbaru dari permainan Anda. Jika proses unggah berhasil, UDP kembali pesan “upload berturut-turut” dengan kode status 200. Jika proses unsuccessful, UDP mengembalikan pesan kesalahan dengan kode status non-2xx.

Anda dapat mengunggah keduanya:

  • File APK, file OBB utama (opsional) dan file OBB patch (opsional). Login Ini menimpa file AAB yang ada di konsol UDP.
    This overwrites any existing AAB file in the UDP console.
  • AAB file.
    Ini menimpa file APK atau OBB yang ada di konsol UDP.

Anda hanya dapat meng-upload satu file biner pada suatu waktu; Anda tidak dapat menentukan beberapa file biner untuk perintah. Jika Anda meng-upload jenis file biner yang sama beberapa kali, file terbaru menggantikan yang sebelumnya.

Prerequisites

Untuk menggunakan API UDP, Anda perlu:

  • Token otentikasi yang valid dari konsol UDP. Login Untuk menghasilkan token otentikasi di konsol UDP, buka halaman Developer API dan pilih
    .New Token.
  • Alat untuk mengirim pesan HTTPS POST. Unity merekomendasikan menggunakan curl.

Unggah file biner

Untuk mengunggah file biner ke konsol UDP menggunakan API UDP, gunakan metode HTTP POST, dan atur Content-Type ke multipart/form-data. Untuk informasi pada parameter API yang didukung, lihat API.

Unggah file APK

Untuk mengunggah file APK, tentukan path ke file APK Anda dalam metode POST dengan kunci formulir uploadFile.

Contoh:

$ curl -XPOST -F 'whatsNew="example"' -F 'useGoogleService=false'  -F 'uploadFile=@/path/to/your/APK/example.apk' 
https://distribute.dashboard.unity.com/developer/api/upload/12369536319506/1d1a4cxq774MzPKwsiHgXA?token=NWQxMWIzZmYyNjk0Njc2NGYwMDU0ZTA3ZTk3YjA2ZDA=

Unggah file AAB

Untuk mengunggah file AAB, tentukan path ke file AAB Anda dengan tombol formulir uploadFile.

Contoh:

$ curl -XPOST -F 'whatsNew="example"' -F 'useGoogleService=true  -F 'uploadFile=@/path/to/your/AAB/example.aab' 
https://distribute.dashboard.unity.com/developer/api/upload/12369536319506/1d1a4cxq774MzPKwsiHgXA?token=NWQxMWIzZmYyNjk0Njc2NGYwMDU0ZTA3ZTk3YjA2ZDA=

Unggah file OBB

Untuk mengunggah file OBB:

  1. Tentukan path ke file OBB Anda dengan kunci formulir uploadFile.
  2. Tentukan tipe OBB. Mengatur parameter query kunci obbType ke mainObb atau patchObb.

Contoh unggah file OBB utama:

$ curl -XPOST -F 'uploadFile=@/path/to/your/OBB/exampleMain.obb'
https://distribute.dashboard.unity.com/developer/api/upload/12369536319506/1d1a4cxq774MzPKwsiHgXA?token=NWQxMWIzZmYyNjk0Njc2NGYwMDU0ZTA3ZTk3YjA2ZDA=&obbType=mainObb

Contoh patch Unggulan file OBB:

$ curl -XPOST -F 'uploadFile=@/path/to/your/OBB/examplePatch.obb'
https://distribute.dashboard.unity.com/developer/api/upload/12369536319506/1d1a4cxq774MzPKwsiHgXA?token=NWQxMWIzZmYyNjk0Njc2NGYwMDU0ZTA3ZTk3YjA2ZDA=&obbType=patchObb

info

Sebelum Anda dapat mempublikasikan permainan Anda ke toko di konsol UDP, ikuti langkah-langkah ini:

  1. Edit informasi permainan Anda, termasuk:
    1. Periksa bagian Pengujian Sandbox untuk memastikan pengujian berhasil.
    2. Unggah aplikasi Anda menandatangani kunci pribadi
    3. Menemukan penggunaan iklan Anda.
    4. Mengatur harga premium (jika berlaku).
    5. Lokalisasi informasi game Anda untuk bahasa tambahan (jika ada).
    6. Memutar produk IAP Anda.
  2. Lepaskan revisi permainan Anda.

Anda kemudian dapat melanjutkan untuk mempublikasikan permainan Anda.

Lepaskan revisi Anda

Untuk dapat mempublikasikan permainan Anda ke toko, Anda perlu melepaskannya. Ketika Anda telah menyimpan draft revisi Anda dan siap menerbitkannya:

  1. Di halaman Game Info, pilih RELEASE.
    Jendela terbuka untuk memasukkan informasi rilis.
  2. Masukkan Release notes dan Release label untuk melacak revisi Anda.
  3. Pilih RELEASE.

Penyediaan dirilis versi permainan Anda. Ini memungkinkan Anda menyimpan catatan perubahan pengiriman toko aplikasi Anda sepanjang waktu.

Komponen kunci revisi adalah:

  • build game Anda (file APK)
  • Katalog IAP
  • metadata permainan Anda

Jika Anda mengubah salah satu elemen ini, perubahan pengiriman toko Anda, dan Anda harus membuat revisi baru. Sistem revisi adalah incremental (+1 setiap kali) dan dikelola oleh UDP. Anda tidak dapat membuat nomor revisi kustom. Gunakan catatan revisi dan label ketika Anda melepaskan revisi baru untuk membantu Anda melacak revisi.

Note: Jika revisi draft telah disimpan di konsol UDP tetapi tidak dirilis, Anda tidak dapat mendorong perubahan dari Editor yang akan meningkatkan jumlah revisi.

Mengedit revisi

Untuk menulis draft revisi yang ada.

  1. Pilih revisi yang ada yang ingin Anda edit. draft baru Anda akan mengherankan semua data Informasi Game dari revisi ini.
  2. Pilih Edit untuk membuat perubahan Anda.
  3. Simpan perubahan Anda untuk membuat draft revisi.
  4. Untuk membuat draft revisi terbaru, pilih RELEASE.
    1. Masukkan catatan rilis dan label rilis untuk melacak revisi Anda. Login Perubahan jumlah revisi oleh salah satu dari revisi dirilis tertinggi. Ini menjadi Draft Revisi terbaru
      The revision number increments by one from the highest released revision. This becomes the latest Revision Draft

Menerbitkan permainan Anda ke toko

Gunakan panel Publish untuk mempublikasikan permainan Anda ke toko. Anda hanya dapat mempublikasikan revisi yang dirilis ke toko.

Ikuti langkah-langkah ini untuk setiap toko yang ingin Anda kirimkan permainan Anda ke:

  1. Daftar dengan toko.
  2. Daftarkan permainan Anda dengan toko.
  3. Mengirimkan permainan Anda ke toko.
  4. Pengaturan Lanjutan.
  5. Tambahkan informasi perusahaan.
  6. Ketika setup Anda selesai untuk semua toko, pilih Publish.

Untuk membantu penerbitan ke toko tertentu, lihat Documentation > Partner Store Guides di konsol UDP.

Ketika Anda telah menerbitkan permainan Anda, Anda dapat memantau kinerjanya di Reporting Dashboard.

Daftar dengan toko

Untuk dapat mempublikasikan permainan Anda ke toko, Anda perlu memiliki akun dengan mereka.

Untuk membuat akun dengan toko:

  1. Di halaman Publish, pilih Daftar ke nama <store>.
    Untuk beberapa toko, Anda mungkin diarahkan di luar konsol UDP untuk menyelesaikan proses pendaftaran Anda dengan toko.
  2. Ikuti proses onboarding untuk toko.

Setelah Anda mendaftar di toko, Anda dapat daftarkan permainan Anda dengannya.

Sitemap Hanya Pemilik Organisasi dan Manajer dapat mendaftar di toko. Lihat Note: untuk rincian lebih lanjut.Izin terkait organisasi for more details.

Simpan akun per organisasi Unity. Game apa pun yang Anda publikasikan melalui UDP dilampirkan ke Organisasi dari mana Anda dipublikasikan. Jika Anda mengakses konsol UDP di bawah organisasi yang berbeda, Anda perlu mendaftar lagi dan membuat akun yang berbeda. Untuk informasi lebih lanjut tentang cara mendaftar ke toko individu, lihat Documentation > Partner Store Guides di konsol UDP.

Daftarkan permainan Anda dengan toko

Ketika Anda telah mendaftar ke toko, Anda dapat mendaftarkan permainan Anda dengan toko.

  1. Di halaman Publikasi, pilih Daftarkan permainan Anda dengan nama <store>.
  2. Konfirmasikan nama paket yang ingin Anda daftarkan dengan toko dan pilih REGISTER. Beberapa toko meminta informasi tambahan ketika mendaftarkan permainan.

Ketika permainan Anda terdaftar di toko, Anda tidak bisa lagi mengubah nama paket untuk toko itu.

Jika Anda menggunakan UDP untuk menghasilkan kunci pribadi Pendaftaran aplikasi, UDP menghasilkan kunci khusus toko untuk menandatangani build yang dikemas ulang. Ini juga dapat mempengaruhi layanan pihak ketiga yang terintegrasi dalam permainan Anda. Sertifikat khusus toko tersedia di bagian Advanced ketika permainan Anda telah dibongkar.

Mendaftarkan build yang Anda repacked dengan cara ini membuat permainan Anda lebih rentan terhadap peringatan Google Play Protect.

Mengirimkan permainan Anda ke toko

Di tab Publish, pilih target langkah untuk toko yang diberikan.

  1. Untuk mempublikasikan permainan Anda ke toko untuk pertama kalinya, pilih kotak centang untuk toko.
  2. Pilih Submit to Store.
    UDP repacks dan menyerahkan permainan Anda ke toko.

UDP membuat build terpisah untuk setiap toko yang Anda publikasikan, hanya berisi SDK khusus toko yang diperlukan. Setiap build khusus toko ditandatangani dengan sertifikat UDP yang spesifik untuk permainan Anda dan untuk setiap toko. Anda dapat menemukan dan mengambil sertifikat dari bagian Lanjutan setiap toko.

Selain itu, UDP menambahkan suffix khusus untuk nama paket, jika berlaku. Jika ini diperlukan, konsol UDP akan menampilkan informasi ini selama langkah Register.

Note: Jika diperlukan, di halaman Publikasi, Anda dapat mengkonfigurasi:

  • Negara-negara yang ingin menerbitkan permainan Anda untuk
  • Pengaturan lanjutan untuk permainan Anda

Mengirimkan revisi yang berbeda ke toko yang berbeda

Anda hanya dapat mengirimkan revisi terbaru ke toko. Jika Anda ingin mengirimkan revisi yang berbeda ke toko yang berbeda, Unity menyarankan Anda menggunakan label rilis dan catatan untuk mengidentifikasi revisi Anda.

Contoh: Anda ingin mengirimkan satu revisi untuk StoreA dan revisi lain untuk TokosBCD.

  1. Buat revisi Anda (misalnya, revisi 1) untuk StoreA. Ketika melepaskan revisi 1, tulis catatan rilis yang jelas dan label rilis Anda StoreA.
    1. Kirim revisi 1 ke StoreA.
  2. Buat revisi baru untuk TokosBCD. draft revisi ini akhirnya akan menjadi revisi 2. Ketika melepaskan revisi 2, tulis catatan rilis yang jelas dan label rilis Anda StoresBCD.
  3. Kirim revisi 2 ke StoresBCD.
  4. Ketika Anda perlu mengirimkan rilis baru ke StoreA, Anda dapat melakukannya berdasarkan pengajuan Anda sebelumnya ke toko itu. Label rilis Anda akan menunjukkan bahwa revisi 1 adalah revisi terakhir yang dilabeli StoreA.
  5. Pilih revisi 1, dan edit. Ini akan membuat draft revisi berdasarkan revisi 1. Ketika Anda melepaskannya, itu akan menjadi revisi 3 (sebagai revisi 2 telah dirilis).
  6. Ketika melepaskan revisi 3, terus menggunakan label StoreA. Kirim revisi 3 ke StoreA.

Repacking permainan Anda

Jika Anda sebelumnya telah mengirimkan permainan Anda ke toko, Anda dapat membongkar permainan tanpa mengunggah katalog IAP baru. Ini menggunakan katalog IAP yang paling baru-baru ini.

  1. Untuk mengirim ulang tanpa mengunggah katalog IAP baru, pilih kotak centang untuk toko.
  2. Pilih Repack Game.

Sebelum Anda mengirimkan build repacked Anda ke toko, unduh APK dari tab Status dan mengujinya untuk memastikan pembelian dalam aplikasi Anda bekerja seperti yang diharapkan di lingkungan komersial toko.

Note: Perubahan yang telah Anda buat dalam pengaturan lanjutan untuk toko tidak terlalu dipungkiri jika Anda mengirimkan versi baru dari permainan Anda.

Menambahkan informasi perusahaan

Untuk mempublikasikan permainan Anda ke toko UDP, Anda perlu membuat profil informasi perusahaan. Toko menampilkan informasi ini di bagian "Tentang pengembang" dari daftar game Anda di toko aplikasi mereka. Anda hanya perlu melakukan ini sekali. Meme it

  1. Di konsol UDP, pilih Settings > Company Info.
  2. Masukkan dan simpan informasi perusahaan Anda.

Menerbitkan permainan Anda

Setelah Anda menyelesaikan semua langkah di atas, menerbitkan permainan Anda.

  1. Di tab Publikasi, pilih setiap toko yang ingin Anda kirimkan revisi permainan ini.
  2. Pilih PUBLISH.

Hanya revisi terbaru dari permainan Anda diambil melalui langkah-langkah target yang dipilih untuk setiap toko.

Jika ada masalah dengan pengajuan Anda, ini ditampilkan di bagian Detail.

Untuk toko-toko yang hanya sebagian terintegrasi dengan UDP, Anda harus menyelesaikan pengajuan konsol pengembang sendiri toko. Untuk melakukan ini, pilih Go to Store di sebelah revisi yang diajukan. Sebuah tooltip menjelaskan langkah-langkah yang masih diperlukan. Ikuti tautan dan lengkapkan pengajuan Anda di konsol toko.

Foto’s

Ketika Anda tidak perlu lagi permainan di konsol UDP, Anda dapat menarsipkannya.

Untuk menyaring permainan Anda:

  1. Pergi ke panel My Games.
  2. Pilih filter Anda untuk permainan. Secara default, Anda dapat melihat semua game aktif.
  3. Selebihnya permainan untuk menampilkan menu More (Perusahaan).
  4. Pilih Archive.

Ketika Anda telah mengarsipkan permainan, Anda kemudian dapat mengembalikannya atau menghapusnya dari konsol UDP.

Deleting permainan

Untuk menghapus permainan Anda tidak perlu lagi di konsol UDP:

  1. Ikuti langkah ke arsip permainan.
  2. Selebihnya permainan untuk menampilkan menu More (Perusahaan).
    1. Pilih Delete.

Note: Ini secara permanen menghilangkan permainan dari tab My Games konsol UDP, dan semua data game dari Dasbor Pelaporan.

Restoring permainan arsip

Untuk mengembalikan permainan arsip:

  1. Pergi ke panel My Games.
  2. Pilih filter archived dalam kriteria seleksi.
  3. Pada permainan Anda ingin mengembalikan, pilih menu More (Perusahaan).
  4. Pilih Restore.
