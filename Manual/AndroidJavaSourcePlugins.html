Menggunakan file sumber Java atau Kotlin sebagai plug-ins

Unity mendukung plug-insSatu set kode yang dibuat di luar Unity yang menciptakan fungsi dalam Unity. Ada dua jenis plug-ins yang dapat Anda gunakan di Unity: Managed plug-ins (diproduksi. Rakitan NET dibuat dengan alat-alat seperti Studio Visual) dan plug-ins asli (pustaka kode asli yang spesifik platform). More info
Lihat di Glossary untuk Android ditulis di Java atau Kotlin. Fungsi ini memungkinkan Anda untuk menambahkan file .java atau .kt ke dalam Proyek Unity Anda dan mereka dikompilasi ketika Anda membangun Pemain Android. Sebelumnya, Anda harus precompile file ini sebagai perpustakaan Android.

Untuk mengkompilasi file sumber sebagai plug-in, ikuti langkah-langkah ini:

  1. Buka Editor Unity.
  2. Pilih Proyek untuk menambahkan plug-in ke.
  3. Di jendela Proyek, pilih folder Aset dan buat folder untuk plug-in Anda (Anda dapat menjatuhkan langsung ke Aset, tetapi untuk proyek nyata lebih baik menggunakan subfolder).
  4. Seret dan lepaskan file sumber Anda ke folder.
  5. Pilih masing-masing file.
  6. Di jendela inspectorJendela Unity yang menampilkan informasi tentang Pengaturan GameObject yang dipilih saat ini, aset atau proyek, memungkinkan Anda untuk memeriksa dan mengedit nilai. More info
    Lihat di Glossary    , di bawah Select platforms for plugin, verifikasi bahwa hanya Android dipilih.
  7. Membangun Proyek Anda.

Note: Anda dapat menempatkan file sumber Anda di folder mana pun di Proyek Anda kecuali di lokasi penggunaan khusus seperti StreamingAssets. Jika Anda menempatkan file di lokasi ini, Editor Unity tidak akan menampilkan inspektur plug-in. Untuk informasi tambahan, lihat AAR plug-ins dan Android Perpustakaan.

Gunakan kelas Android SDK untuk memanggil metode di plug-in Anda. Untuk informasi tambahan lihat, “Menggunakan plug-in Java Anda dari C# scriptsSepotong kode yang memungkinkan Anda untuk membuat Komponen Anda sendiri, memicu peristiwa permainan, memodifikasi sifat komponen dari waktu ke waktu dan menanggapi input pengguna dengan cara apa pun yang Anda sukai. More info
Lihat di Glossary dengan kelas pembantu” di JAR plug-ins.
