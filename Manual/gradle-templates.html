Templat Gradle

Templat Gradle mengkonfigurasi cara membangun aplikasi Android menggunakan GradleSistem build Android yang mengotomatiskan beberapa proses build. Otomasi ini berarti bahwa banyak kesalahan build umum cenderung terjadi. More info
Lihat di Glossary. Setiap template Gradle mewakili proyek Gradle tunggal. Proyek Gradle dapat mencakup, dan tergantung pada, proyek Gradle lainnya.

File template

Templat Gradle terdiri dari file berikut:

File Location Contains
baseProjectTemplate.gradle Dalam proyek yang diekspor, folder root/build.gradle folder Informasi konfigurasi yang mempengaruhi semua modul dalam proyek Final Gradle. Ini menentukan versi Plugin Gradle Android untuk menggunakan dan lokasi plugin java. Lokasi adalah kombinasi repositori online dan plugin java di dalam proyek ini.
launcherTemplate.gradle_ Dalam proyek yang diekspor, folder root/launcher/build.gradle folder Instruksi tentang cara membangun aplikasi Android. Ini termasuk bundling, signing, dan apakah untuk membagi apkFormat paket Android output oleh Unity. Sebuah APK secara otomatis digunakan untuk perangkat Anda ketika Anda memilih File > Build & Run. More info
Lihat di Glossary. Ini tergantung pada proyek kesatuanLibrary dan output baik file .apk atau bundel aplikasi.
mainTemplate.gradle Dalam proyek yang diekspor, folder root/unityLibrary/build.gradle folder Instruksi tentang cara membangun Unity sebagai Perpustakaan. Ini menghasilkan file .aar. Anda dapat menimpa template Unity dengan template kustom di Editor Unity. Lihat bagian template build Gradle kustom di halaman ini untuk rincian lebih lanjut.
libTemplate.gradle Varies Jika Proyek Perpustakaan Android plug-inSatu set kode yang dibuat di luar Unity yang menciptakan fungsi dalam Unity. Ada dua jenis plug-ins yang dapat Anda gunakan di Unity: Managed plug-ins (diproduksi. Rakitan NET dibuat dengan alat-alat seperti Studio Visual) dan plug-ins asli (pustaka kode asli yang spesifik platform). More info
Lihat di Glossary tidak termasuk file build.gradle, Unity menggunakan file libTemplate.gradle sebagai template untuk menghasilkan satu. Setelah Unity menghasilkan file build.gradle, atau jika salah satu yang sudah ada di direktori plug-in, Unity menyalin plug-in ke proyek Gradle.

Menggunakan template Gradle kustom

Ada dua file .gradle yang mengontrol proses build Gradle untuk modul kesatuanLibrary:

  • build.gradle: Tentukan instruksi build.
  • settings.gradle: Mengandung nama modul yang sistem build Gradle harus mencakup ketika membangun proyek. komponen proyek yang instruksi dalam file build.gradle digunakan dalam proses build.

Login Login

File build.gradle mengandung variabel template yang menentukan instruksi build.

Secara default, Unity menggunakan file mainTemplate.gradle dari direktori instalasi Unity untuk membuat file build.gradle untuk Login Database. Untuk membuat file mainTemplate.gradle Anda sendiri:

  1. Buka jendela Pengaturan Proyek (menu: Edit > Project Settings).
  2. Pilih tab Player kemudian buka Pengaturan Pemain Android:
    %
  3. Pada bagian Publishing Settings, aktifkan Custom Main Gradle Template. Ini menciptakan file template Gradle yang disebut mainTemplate.gradle dan menampilkan path ke file. Unity sekarang menggunakan file mainTemplate.gradle ini untuk membuat file build.gradle. Untuk daftar variabel template dan deskripsi apa yang masing-masing, lihat variabel template.

variabel template

File settings.gradle mengandung komponen proyek yang digunakan dalam proses build.

Secara default, Unity menggunakan file settingsTemplate.gradle dari direktori instalasi Unity untuk membuat file settings.gradle untuk membangun Anda. Untuk membuat file settingsTemplate.gradle Anda sendiri, buat file settingsTemplate.gradle di folder Assets/Plugins/Android/ proyek Anda. Ini menimpa template default.

If Anda membuat file settings.gradle kustom, menyadari hal berikut:

  • Unity menciptakan komponen launcher dan unityLibrary secara default dan Anda harus menyertakannya dalam file settingsTemplate.gradle Anda. Untuk melakukan ini, tambahkan include ':launcher', ':unityLibrary' sebagai entri ke settingsTemplate.gradle Anda.
  • Jika Anda menggunakan plug-ins Perpustakaan Android, Unity termasuk dalam file settings.gradle akhir dengan mengganti entri **INCLUDES**. Ini berarti Anda harus menambahkan **INCLUDES** sebagai entri ke settingsTemplate.gradle Anda.

Modifikasi proyek Gradle yang diekspor menggunakan C#

Untuk memodifikasi proyek Gradle setelah Unity merakitnya, menciptakan kelas yang mengintip dari IPostGenerateGradleAndroidProject dan menimpa fungsi Android SDK. Fungsi ini menerima jalan ke modul kesatuanLibrary sebagai parameter dan Anda dapat menggunakannya untuk mencapai manifestasi aplikasi dan sumber daya melalui skrip C#.

Warning: Unity sekarang menggunakan pipa build inkremental yang berarti Mempercepat proyek Gradle yang sama untuk membangun berturut-turut. Ini berarti bahwa setiap modifikasi yang Anda lakukan menggunakan API ini menumpuk sejak Unity tidak lagi membuat proyek Gradle baru untuk setiap build. Misalnya, jika Anda menggunakan API ini untuk menambahkan file tambahan ke proyek Gradle, pekerjaan build pertama seperti yang diharapkan, tetapi selama kedua membangun file sudah ada. Contoh kedua adalah jika Anda menggunakan API ini untuk menambahkan izin ke file tertentu. Setiap build berturut-turut menambahkan entri lain untuk izin. Sangat penting untuk memastikan bahwa modifikasi yang Anda inginkan untuk membuat tidak ada yang ada dalam membangun.
