Pemecahan masalah Gradle

Jika Anda hanya beralih ke ekspor proyek Android Anda menggunakan GradleSistem build Android yang mengotomatiskan beberapa proses build. Otomasi ini berarti bahwa banyak kesalahan build umum cenderung terjadi. More info
Lihat di Glossary bukan sistem lama, Anda dapat menemukan kesalahan build, terutama jika Anda menggunakan perpustakaan Android tambahan, atau jika Anda telah menambahkan AndroidManifest.xml kustom.

The Android Gradle plug-inSatu set kode yang dibuat di luar Unity yang menciptakan fungsi dalam Unity. Ada dua jenis plug-ins yang dapat Anda gunakan di Unity: Managed plug-ins (diproduksi. Rakitan NET dibuat dengan alat-alat seperti Studio Visual) dan plug-ins asli (pustaka kode asli yang spesifik platform). More info
Lihat di Glossary jauh lebih pemilih daripada sistem ADT / Ant lama. Tidak menerima apa pun yang mempertimbangkan kesalahan, apakah itu duplikat simbol, referensi ke sumber daya yang tidak ada, atau proyek perpustakaan yang menetapkan atribut yang sama sebagai aplikasi utama.

Dalam kebanyakan kasus, memperbaiki masalah melibatkan mengedit file AndroidManifest.xml; baik yang utama, atau satu dari perpustakaan penggunaan proyek Anda.

Dalam proyek non-trivial, atau jika proyek memiliki masalah yang tidak dijelaskan oleh bagian pemecahan masalah di bawah ini, ekspor proyek sebagai proyek Gradle (dari Build Settings) dan membangun dari baris perintah. Membangun dari baris perintah memberi Anda pesan kesalahan yang lebih rinci, dan membuat giliran yang lebih cepat saat menerapkan perubahan.

Masalah spesifik

Sumber daya tidak ditemukan

File AndroidManifest.xml, baik yang utama atau dalam perpustakaan, referensi sumber daya yang tidak ada. Sering itu adalah ikon aplikasi atau string label yang ditetapkan oleh perpustakaan. Hal ini dapat terjadi jika Anda telah menyalin manifestasi utama Anda ke proyek perpustakaan tanpa menghapus referensi tersebut.

Hapus atribut dari salah satu dari Android App Manifests - biasanya yang dari perpustakaan.

MinSDK di Manifest

Atribut android:minSdkVersion ditentukan dalam file AndroidManifest.xml. Ini dapat dalam file manifestasi utama, atau dalam file di salah satu direktori yang memperlakukan Unity sebagai perpustakaan Android. .aar plugin tidak boleh menyebabkan masalah ini.

Solusinya adalah menghapus elemen uses-sdk android:minSdkVersion dari manifestasi utama dan / atau manifestasi di direktori perpustakaan. Sebaliknya, Anda harus menentukan versi SDK minimum dalam file build.gradle. Jika Anda tidak memiliki template Gradle kustom, Unity secara otomatis menangani ini. Jika Anda memiliki template Gradle kustom, pastikan minSDK ditentukan di bagian defaultConfig template.

File Duplikat di APK

Anda memiliki nama file collisiontabrakan terjadi ketika mesin fisika mendeteksi bahwa tabrakan dari dua GameObjects membuat kontak atau tumpang tindih, ketika setidaknya satu memiliki komponen kaku dan bergerak. More info
Lihat di Glossary antara aplikasi utama Anda dan proyek perpustakaan, atau antara dua proyek perpustakaan. Perlu diingat bahwa semua file disalin ke paket APKFormat paket Android output oleh Unity. Sebuah APK secara otomatis digunakan untuk perangkat Anda ketika Anda memilih File > Build & Run. More info
Lihat di Glossary yang sama.

Anda perlu menghapus salah satu file.

Nama paket Colliding

Perpustakaan tidak dapat menggunakan paket Java yang sama sebagai aplikasi utama, atau perpustakaan lainnya.

Biasanya, Anda harus mengubah nama paket perpustakaan ke sesuatu yang berbeda. Jika perpustakaan mengandung banyak kode, mungkin lebih mudah untuk mengubah nama paket utama (dari pengaturan Player).

Colliding attributes

Perpustakaan tidak dapat menahan atribut dari file AndroidManifest.xml utama. Seringkali kesalahan ini disebabkan oleh pengaturan perpustakaan ikon aplikasi atau string label, mirip dengan masalah Resource not found di atas.

Baik menghapus atribut dari perpustakaan, atau tambahkan atribut tools:replace ke tag application Anda, untuk menunjukkan bagaimana konflik merge harus diselesaikan.
