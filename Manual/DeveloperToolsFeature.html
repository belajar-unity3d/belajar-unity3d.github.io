Getting started

Dukungan unity IDE berikut: Visual Studio (Windows, macOS), Visual Studio Code (Windows, macOS, Linux) dan JetBrains Rider (Windows, macOS, Linux).

Untuk memilih IDE mana yang digunakan oleh Editor Unity, pergi ke Unity /Edit > Preferences > External Tools > External Script Editor.

Kerangka Uji Unity (UTF) memungkinkan Anda menguji kode Anda dalam Mode Edit dan Mode Play, dan juga pada platform target seperti Standalone, Android dan iOS.

Untuk mengakses Kerangka Uji Unity (UTF) di Editor Unity, pergi ke Window > General > Test Runner.

Profil Analyzer agregat dan memvisualisasikan bingkai dan data penanda dari satu set bingkai Unity ProfilerJendela yang membantu Anda untuk mengoptimalkan permainan Anda. Ini menunjukkan berapa banyak waktu yang dihabiskan di berbagai bidang permainan Anda. Sebagai contoh, dapat melaporkan persentase waktu yang dihabiskan rendering, aimating, atau dalam logika permainan Anda. More info
Lihat di Glossary untuk membantu Anda memahami perilaku mereka. Anda dapat menggunakan Analisis Profil untuk membandingkan dua set sisi data, yang melengkapi analisis bingkai tunggal yang sudah tersedia dalam Profiler Unity.

Untuk membuka jendela Profil Analyzer, pergi ke Window > Analysis > Profile Analyzer.

Gunakan paket Penutup Kode dengan Pelari Uji untuk mengumpulkan dan menyajikan informasi cakupan pengujian. Jalankan tes Anda dengan cakupan kode diaktifkan untuk melihat garis kode mana yang dieksekusi ketika tes berjalan, dan apakah tes lulus atau gagal.

Untuk membuka jendela Penutup Kode di Editor Unity, buka Window > Analysis > Code Coverage.

Gunakan paket Coroutines Editor untuk memulai eksekusi metode iterator dalam Editor, mirip dengan bagaimana kita menangani Coroutines di dalam MonoBehaviour% ruangelemen selama runtime.scriptsSepotong kode yang memungkinkan Anda untuk membuat Komponen Anda sendiri, memicu peristiwa permainan, memodifikasi sifat komponen dari waktu ke waktu dan menanggapi input pengguna dengan cara apa pun yang Anda sukai.
