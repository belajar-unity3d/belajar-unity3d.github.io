Pemecahan Masalah Editor

Bagian berikut menjelaskan bagaimana memecahkan masalah dan mencegah masalah dengan editor Unity dalam situasi yang berbeda. Secara umum, pastikan komputer Anda memenuhi semua, up-to-date, dan Anda memiliki izin pengguna yang diperlukan dalam sistem Anda. Juga membuat cadangan secara teratur untuk melindungi proyek Anda.

Versions

Anda dapat menginstal berbagai versi editor di folder yang berbeda. Namun, pastikan Anda backup proyek Anda karena ini mungkin ditingkatkan dengan versi yang lebih baru, dan Anda tidak akan dapat membukanya dalam versi Unity yang lebih tua. Lihat halaman manual pada memasang Unity untuk informasi lebih lanjut.

Lisensi add-on hanya berlaku untuk versi Unity yang berbagi nomor utama yang sama, misalnya 3.x dan 4.x. Jika Anda meningkatkan ke versi kecil Unity, misalnya 4.0 ke 4.1, add-on akan disimpan.

Activation

Aktivasi Internet adalah metode yang disukai untuk menghasilkan lisensi Unity Anda. Tetapi jika Anda mengalami masalah mengikuti langkah-langkah ini:

  1. Lepaskan komputer Anda dari jaringan, jika Anda mungkin mendapatkan kesalahan "tx_id tidak valid".
  2. Pilih Aktivasi Manual.
  3. Klik Save License Request.
  4. Pilih lokasi simpan yang diketahui, misalnya folder Unduhan.
  5. Rekoneksi ke jaringan dan buka https://license.unity3d.com/
  6. Di kolom file klik Jelajahi, dan pilih file permintaan lisensi.
  7. Pilih lisensi yang diperlukan untuk Unity dan isi informasi yang diminta.
  8. Klik Download Lisensi dan menyimpan file.
  9. Kembali ke Unity dan pilih Aktivasi Manual jika diperlukan.
  10. Klik pada Baca Lisensi dan kemudian pilih file lisensi yang diunduh.

Jika Anda masih memiliki masalah dengan mendaftar atau masuk ke akun pengguna Anda silahkan hubungi.

Kegagalan untuk Mulai

Jika Unity crash ketika memulai kemudian pertama-tama pastikan bahwa komputer Anda memenuhi minimal. Juga update ke driver grafis dan suara terbaru.

Jika Anda mendapatkan kesalahan menulis disk, Anda harus memeriksa pembatasan akun pengguna Anda. Ketika di MacOS, perhatikan "pengguna akar" tidak dianjurkan dan belum diuji pada mode ini. Unity harus selalu memiliki izin menulis untuk foldernya, tetapi jika Anda memberi mereka secara manual memeriksa folder ini:

Di Windows:

  • folder instalasi Unity
  • %AllUsersProfile%\Unity (biasanya C:\ProgramData\Unity)
  • C:\Documents and Settings\<user>\Local Settings\Application Data\Unity
  • C:\Users\<user>\AppData\Local\Unity

Login:

  • Konten paket Unity.app
  • /Library/Application Support/Unity
  • ~/Library/Logs/Unity

Beberapa pengguna mengalami kesulitan ketika menggunakan hard disk diformat dengan partisi non-natif, dan menggunakan perangkat lunak tertentu untuk menerjemahkan data antara perangkat penyimpanan.

Fonts

Korrupt font dapat crash Unity, Anda dapat menemukan file yang rusak setelah langkah-langkah ini:

Di Windows:

  1. Buka folder font pada komputer Anda, terletak di folder "Windows".
  2. Pilih "Details" dari menu "View".
  3. Periksa kolom "Ukuran" untuk font dengan ukuran "0", yang menunjukkan file yang bermasalah.
  4. Hapus font korup dan instal ulang mereka.

Di MacOS:

  1. Luncurkan aplikasi Buku Font Anda.
  2. Pilih semua font.
  3. Buka menu "File" dan pilih "Validate Fonts" -> font bermasalah akan ditampilkan sebagai tidak valid.
  4. Hapus font korup dan instal ulang mereka.

Sistem mungkin memiliki sumber daya yang terlatih, misalnya berjalan dalam mesin virtual. Gunakan Task Manager untuk menemukan proses mengkonsumsi banyak memori.

Proyek Korupsi atau Instalasi

Unity dapat mencoba membuka proyek yang rusak, ini mungkin termasuk proyek sampel default. Dalam hal ini nama atau memindahkan folder proyek. Setelah Unity dimulai dengan benar Anda dapat mengembalikan folder proyek jika ingin.

Dalam hal instalasi korup, Anda mungkin perlu menginstal ulang Unity - melihat instruksi di bawah ini.

Di Windows, ada masalah seperti kesalahan instalasi, korupsi registry, konflik, dll. Sebagai contoh, error 0xC0000005 berarti program telah mencoba untuk mengakses memori yang seharusnya tidak. Jika Anda menambahkan perangkat keras atau driver baru-baru ini, lepaskan dan mengganti perangkat keras untuk menentukan apakah itu menyebabkan masalah. Menjalankan perangkat lunak diagnostik dan memeriksa informasi tentang memecahkan masalah sistem operasi.

Kinerja dan Kecelakaan

Jika editor berjalan perlahan atau crash, terutama pada build, ini mungkin disebabkan oleh semua sumber daya sistem yang tersedia dikonsumsi. Tutup semua aplikasi lain ketika Anda membangun proyek. Bersihkan sistem menggunakan utilitasnya, dan konsultasikan Task Manager (Windows) atau Activity Monitor (MacOS) untuk mengetahui apakah ada proses yang menggunakan banyak sumber daya, misalnya memori. Terkadang perangkat lunak perlindungan virus dapat memperlambat atau bahkan memblokir sistem file dengan proses pemindaiannya.

Rugi Proyek

Ada banyak faktor yang dapat menghancurkan proyek, Anda harus constantly backup your projects untuk mencegah kecelakaan yang sayang. Ketika di MacOS, aktifkan TimeMachine menggunakan hard disk eksternal yang disediakan untuk tujuan tunggal ini. Setelah kehilangan Anda dapat mencoba utilitas pemulihan file yang ada, tetapi kadang-kadang ini tidak dapat dibalik.

Re-instalasi

Ikuti langkah-langkah ini untuk menginstal ulang editor:

  1. Uninstall Unity. Ketika di MacOS, tarik aplikasi Unity untuk sampah.

  2. Hapus file ini jika ada:

    • Windows:
      • %AllUsersProfile%\Unity\ (biasanya C:\ProgramData\Unity)
    • MacOS:
      • /Library/Application Support/Unity/

  3. Restart komputer.

  4. Unduh versi terbaru dari situs web kami, karena instalasi asli Anda mungkin rusak: http://unity3d.com/unity/download/archive

  5. Reinstall Unity.
