Memasang plugin SCM Plastik untuk Unity

Plugin ini tersedia melalui Manajer Paket Unity.

Anda harus menghapus plugin Plastik SCM Asset StoreSebuah perpustakaan yang berkembang dari aset gratis dan komersial yang diciptakan oleh Unity dan anggota komunitas. Menawarkan berbagai aset, dari tekstur, model dan animasi ke seluruh contoh proyek, tutorial dan ekstensi Editor. More info
Lihat di Glossary dari proyek Unity Anda untuk menggunakan Plastic SCM untuk Unity melalui paket Version ControlSistem untuk mengelola perubahan file. Anda dapat menggunakan Unity bersama dengan alat kontrol versi yang paling umum, termasuk Perforce, Git, Mercurial dan PlasticSCM. More info
Lihat di Glossary. Berikut adalah langkah-langkah untuk melakukannya:

  1. Navigasi ke folder Aset / Plasma Anda di Proyek Unity Anda. Hapus folder PlasticSCM.
  2. Buka kembali Proyek Unity Anda untuk menampilkan ulang paket Anda.

Note: Folder ruang kerja .plastic dalam akar Proyek Unity tidak perlu dihapus karena dapat dibaca oleh paket Kontrol Versi.

Anda dapat mengakses SCM Plastik untuk Unity pada versi yang didukung oleh navigasi ke Window > Plastic SCM. Dari sana, Anda akan berjalan melalui proses onboarding sederhana, yang akan memandu pengaturan akun Anda dan memastikan bahwa proyek Unity Anda dikonfigurasi dengan benar.

Menggunakan plugin SCM Plastik

Plugin SCM Plastik memungkinkan jendela Plastic SCM di Editor Unity. Gunakan jendela Plastic SCM untuk mengelola perubahan file Anda, termasuk:

  • Tautan proyek Anda ke kendali sumber
  • Check-in files
  • Tidak ada perubahan
  • Bandingkan revisi file

Jendela Plastic SCM menampilkan daftar file baru dan berubah dalam proyek Anda. Ketika Anda menambahkan file baru, atau menyimpan perubahan ke file yang ada, jendela Plastic SCM secara otomatis menampilkan file yang terkena.

Untuk membuka jendela Plastic SCM, pilih Window > Plastic SCM.

Membuat ruang kerja baru

Untuk menggunakan plugin Plastik SCM pada proyek Unity, Anda harus membuat ruang kerja untuk proyek di jendela Plastic SCM atau di Plastic SCM GUI.

Untuk menciptakan ruang kerja baru untuk proyek Anda di Editor Unity: 1. Pilih Window > Plastic SCM. 2. Tentukan nama repositori, nama ruang kerja dan jalan ke proyek. Secara default, jalur proyek dan nama yang ada digunakan. 3. Pilih apakah Anda ingin menggunakan SCM Plastik sebagai pengembang (untuk fungsi SCM Plastik penuh) atau seniman (untuk alur kerja yang disederhanakan).

Setelah Anda membuat ruang kerja Anda, jendela Plastic SCM menampilkan file proyek Anda. Selain itu, Plastik SCM secara otomatis menciptakan ignore fileFile khusus yang digunakan dalam banyak Sistem Kontrol Versi yang menentukan file yang akan dikecualikan dari ditempatkan di bawah kendali versi. Dalam proyek Unity ada sejumlah file yang bisa dikecualikan dari kontrol versi, dan menggunakan File Ignore adalah cara terbaik untuk mencapai ini. More info
Lihat di Glossary di ruang kerja, yaitu, folder proyek Anda.

Memperkuat file proyek Anda ke kendali sumber

Setelah Anda menghubungkan proyek Anda ke ruang kerja SCM Plastik, tab Pending Changes dari jendela Plastic SCM menampilkan semua file proyek yang ada sebagai Added and private. Untuk menyertakan file ini di ruang kerja, Anda harus periksa mereka.

Setelah Anda memeriksa file Anda, tab Pending changes menyegarkan dan tidak lagi daftar file yang telah Anda periksa. Selain itu, Plastik SCM GUI sekarang menampilkan:

  • File proyek Anda di Workspace Explorer
  • Check-in terbaru Anda dalam tampilan Changeset

Memeriksa file

Untuk memeriksa file:

  1. Pilih Window > Plastic SCM.
  2. Di jendela Plastic SCM, pilih file yang ingin Anda periksa.
  3. Dalam bidang teks, masukkan komentar untuk menjelaskan perubahan Anda.
  4. Pilih Checkin.

Perubahan Anda tercantum dalam tampilan SCM GUI Plastik, dalam tampilan Changesets.

Mengabaikan file

Ketika Anda membuat ruang kerja menggunakan plugin Plastik SCM, secara otomatis membuat file yang mengabaikan di ruang kerja Anda. Ini berisi daftar default dari jenis file dan jalur untuk mengecualikan dari kontrol sumber. Jika ada file lain dalam proyek Anda yang ingin Anda kecualikan dari repositori bersama, tambahkan ke file mengabaikan:

  1. Di jendela Plastic SCM, klik kanan file yang ingin Anda abaikan.
  2. Pilih Add to ignored list.
  3. Pilih nama file.

File yang Anda pilih untuk mengabaikan dihapus dari daftar Pending changes. Anda dapat mengabaikan nama file tertentu, seluruh folder, dan semua file dengan ekstensi tertentu.

Pelajari lebih lanjut tentang mengabaikan file.

Melihat perubahan file

Untuk membandingkan file yang diperbarui dengan versi sebelumnya, klik kanan file di jendela Plastic SCM dan pilih Diff. Ini membuka pandangan Differences, yang membandingkan sumber repositori dengan salinan lokal Anda. Ini adalah tampilan teks.

Mengalihkan perubahan file

Jika Anda telah membuat perubahan dalam proyek Anda yang tidak ingin Anda simpan, Anda dapat kembali kembali ke versi sebelumnya, yaitu, versi checked-in terbaru. Untuk mengubah perubahan file, pilih file di tab Pending changes dan pilih Undo changes.
