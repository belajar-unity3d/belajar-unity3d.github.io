Log files

Unity menghasilkan file log untuk Editor, manajer paket, lisensi, pemain pengembangan, dan Hub. Anda dapat menggunakan file log ini untuk memahami di mana masalah yang terjadi dalam aplikasi Anda.

Unity menambahkan semua pesan, peringatan, dan kesalahan dari Jendela konsolJendela Unity Editor yang menunjukkan kesalahan, peringatan dan pesan lain yang dihasilkan oleh Unity, atau skrip Anda sendiri. More info
Lihat di Glossary ke file log. Untuk menambahkan pesan Anda sendiri ke jendela Konsol, dan log, gunakan Kelas debug.

Setiap sistem operasi menyimpan file log di lokasi yang berbeda. Lokasi default diuraikan di halaman ini, tetapi Anda juga dapat menggunakan argumen baris perintah tertentu untuk mengontrol ketika dan di mana Unity menghasilkan file log. Untuk informasi lebih lanjut, lihat dokumentasi argumen baris perintah.

Lokasi log yang berkaitan dengan editor

Anda dapat mengakses log Editor dari jendela konsol. Untuk melakukan ini, buka Jendela Konsol (menu: Window > General > Console) dan pilih Open Editor Log dari menu jendela Konsol.

Anda dapat mengakses log lain dengan menggunakan aplikasi manajer file sistem operasi Anda.

Linux

Log type Log location
Editor ~/.config/unity3d/Editor.log
Package manager ~/.config/unity3d/upm.log
Licensing client ~/.config/unity3d/Unity/Unity.Licensing.Client.log
Licensing audits ~/.config/unity3d/Unity/Unity.Entitlements.Audit.log

macOS

Di macOS, Anda juga dapat mengakses log Unity melalui Konsol. aplikasi utilitas

Log type Log location

Package manager ~/Library/Logs/Unity/upm.log
Licensing client ~/Library/Logs/Unity/Unity.Licensing.Client.log
Licensing audits ~/Library/Logs/Unity/Unity.Entitlements.Audit.log

Windows

Pada Windows, paket Manager dan editor log ditempatkan di folder yang tidak ditampilkan di Windows Explorer secara default. Untuk melihat folder AppData, Anda harus mengaktifkan Pengaturan Item Tersembunyi di Windows. Untuk informasi lebih lanjut tentang cara melakukan ini, lihat dokumentasi Microsoft pada Lihat file dan folder tersembunyi di Windows.

Pada Windows, aliran keluar standar tidak ada secara default, sehingga Anda harus meluncurkan Editor dengan aliran stdout yang dikonfigurasi yang valid, sebagai proses anak dari sistem CI. Jika Anda menentukan - untuk mengirim output` tostdout`, maka Anda tidak akan melihat output di jendela konsol.

Log type Log location
Editor %LOCALAPPDATA%\Unity\Editor\Editor.log
Package manager %LOCALAPPDATA%\Unity\Editor\upm.log
SYSTEM Sitemap%ALLUSERSPROFILE%\Unity\Editor\upm.log
Licensing client %LOCALAPPDATA%\Unity\Unity.Licensing.Client.log

Crash files %TMP%\CompanyName\ProductName\Crashes

Note: Anda dapat menulis lokasi lokasi folder dengan argumen baris perintah -crash-report-folder.
