Peningkatan ke Unity 3,5

Jika Anda memiliki file FBX dengan root nodePerubahan hierarki animasi yang memungkinkan Unity untuk menetapkan konsistensi antara klip Animasi untuk model generik. Hal ini juga memungkinkan Unity untuk benar campuran antara Animasi yang belum diberi wewenang “di tempat” (yang ada, di mana seluruh Model bergerak posisi dunia sambil memikat). More info
Lihat di Glossary ditandai sebagai kerangka, itu akan diimpor dengan simpul akar tambahan dalam 3,5, dibandingkan dengan 3.4

Unity 3,5 melakukan ini karena ketika mengimpor karakter animasi, setup yang paling umum adalah untuk memiliki satu simpul akar dengan semua tulang di bawah ini dan kerangka di sebelahnya dalam hirarki. Ketika membuat animasi tambahan, itu umum untuk menghilangkan meshGrafik utama primitif Unity. Mesh membuat sebagian besar dunia 3D Anda. Unity mendukung mesh poligon triangulat atau Quadrangulasi. Nurbs, Nurms, permukaan Subdiv harus dikonversi ke poligon. More info
Lihat di Glossary yang dikupas dari file fbx. Dalam hal metode impor baru memastikan bahwa node akar tambahan selalu ada dan dengan demikian animasi dan mesh kulit benar-benar cocok.

Jika koneksi antara instance dan file FBX prefabJenis aset yang memungkinkan Anda untuk menyimpan GameObject lengkap dengan komponen dan properti. Prefab bertindak sebagai template dari mana Anda dapat membuat instance objek baru di tempat kejadian. More info
Lihat di Glossary telah rusak pada 3.4 animasi tidak akan cocok dalam 3,5, dan akibat animasi Anda mungkin tidak bermain.

Dalam hal itu dianjurkan bahwa Anda menciptakan kembali hirarki atau Game Object dengan menyeret file FBX Anda ke dalam sceneAdegan berisi lingkungan dan menu permainan Anda. Pikirkan setiap file Adegan unik sebagai tingkat yang unik. Di setiap Adegan, Anda menempatkan lingkungan, hambatan, dan dekorasi, pada dasarnya merancang dan membangun permainan Anda dalam potongan-potongan. More info
Lihat di Glossary Anda dan mengubahnya.
