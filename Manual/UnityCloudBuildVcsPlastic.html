Menggunakan Dashboard Pengembang Unity untuk mengkonfigurasi Build Unity Cloud untuk Plastik

Unity Cloud BuildLayanan integrasi berkelanjutan untuk proyek Unity yang mengotomatiskan proses pembuatan build pada server Unity. More info
Lihat di Glossary mendukung proyek yang disimpan dalam repositories Plastik SCM.

Note: Fitur ini membutuhkan berlangganan.

Penutup topik ini:

  • Mengkonfigurasi Plastik di Dasbor
  • Menyiapkan platform membangun target

Mengkonfigurasi Plastik di Dasbor

Untuk mengkonfigurasi Cloud Bangun untuk membangun proyek Anda dari repositori plastik:

  1. Masuk ke .
  2. Pada halaman Projects, pilih proyek Anda.
  3. Di jendela dashboard Overview, pada ubin Cloud Build, klik Open Cloud Build Dashboard.
  4. Di jendela Build History, klik Set up Cloud Build.
  5. Di jendela Source Control, klik tab Manual.
  6. Untuk terhubung ke repositori Anda, Anda harus menentukan URL ke server plastik Anda. Di bidang SCM URL, masukkan URL server plastik Anda di kedua format berikut:
    • Login (Jika Anda menghosting server plastik Anda sendiri)
    • WordPress.org (Jika Anda menggunakan Hosting Cloud Plastik)
  7. Pada menu drop-down SCM Types, pilih Plastic SCM.
  8. Klik Next: Access. Ketika Cloud Build terhubung ke situs hosting, secara otomatis mendeteksi apakah repositori Anda adalah publik atau pribadi. Jika repositori Anda adalah publik, Cloud Build secara otomatis terhubung kenya dan Anda dapat melewatkan ke Menyiapkan platform membangun target.
  9. Di jendela Grant us access to your source control:
    1. Masukkan Username dan Password untuk server SCM Plastik Anda.
    2. Pilih Authentication Type dari dropdown.
  10. Pilih Next: Target Setup.

Menyiapkan platform membangun target

Di dasbor, di jendela New Build Target: Basic Info:

  1. Dalam bidang Target Label, masukkan nama untuk membangun.
  2. Dari menu drop-down Repo, pilih repositori dari mana untuk membangun proyek Anda.
  3. Dari menu drop-down Branch, pilih cabang dari mana untuk membangun proyek Anda.
  4. Jika folder akar repositori Anda tidak mengandung aset Anda dan project settingsPengumpulan yang luas dari pengaturan yang memungkinkan Anda untuk mengkonfigurasi bagaimana Fisika, Audio, Jaringan, Grafik, Input dan banyak bidang lain dari jangkauan proyek Anda. More info
    Lihat di Glossary    , di bidang Project Subfolder, masukkan URL folder yang berisi aset dan pengaturan proyek Anda.
  5. Dari menu drop-down Unity Version, pilih versi Unity yang untuk membangun proyek. Untuk memiliki Cloud Build secara otomatis mendeteksi versi Unity proyek Anda, memungkinkan Auto Detect Version. Ini membutuhkan file ProjectSettings/ProjectVersion.txt Anda untuk diperiksa ke kendali sumber.
  6. Jika Anda tidak ingin proyek untuk secara otomatis membangun setiap repositori Anda diperbarui, menonaktifkan toggle Auto-build.
  7. Jika Anda membangun platform iOS atau Android:
  8. Klik Next: Credentials.
  9. Masukkan kredensial dan klik Next: Build.
  10. Untuk semua platform lain, klik Next: Build untuk menyelesaikan konfigurasi dan memulai build awal.
