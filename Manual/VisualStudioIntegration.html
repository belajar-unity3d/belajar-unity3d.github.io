Mengatur Visual Studio sebagai editor skrip default Anda

Unity secara otomatis menggunakan Visual Studio sebagai editor skrip default jika Visual Studio diinstal ketika Anda menginstal Unity, atau jika Anda menginstal Visual Studio sebagai bagian dari proses instalasi Unity. Anda dapat menginstal Visual Studio sebagai modul ke instalasi Unity yang ada. Untuk informasi lebih lanjut, lihat .

Untuk mengatur editor skrip default Anda secara manual:

  • Edit > Preferences.
  • Buka menu External Tools.
  • Klik pada dropdown External Script Editor dan pilih Microsoft Visual Studio. Penampilan perubahan opsi ini tergantung pada versi Microsoft Visual Studio Anda telah diinstal.

A: Menu dropdown Editor Script Eksternal, yang menampilkan nama dan versi editor skrip yang dipilih.

B: Nama dan versi paket Unity yang terintegrasi dengan editor skrip yang dipilih.

Unity menggunakan visual Studio C# compiler untuk mengkompilasi skrip. Ketika Anda menggunakan paket Editor Visual Studio dengan Visual Studio, baik detail tampilan Unity dan Visual Studio dari kesalahan dalam skrip Anda.

Unity secara otomatis membuat dan mempertahankan file Visual Studio.sln dan .csproj. Anda dapat mengontrol apakah Unity menghasilkan file .csproj untuk elemen tertentu dari proyek Anda di menu External Tools di jendela Preferences, seperti di gambar di atas. Aktifkan atau menonaktifkan kotak centang untuk beralih apakah Unity menghasilkan file .csproj untuk opsi tertentu.

Unity mengatur file .sln dan .csproj dalam proyek Anda setiap kali kontributor membuat perubahan pada keadaan file, misalnya, mengedit file yang ada atau membuat yang baru. Anda juga dapat menambahkan file ke solusi Anda dari Visual Studio. Unity mengimpor file baru, dan waktu berikutnya Unity membuat file proyek lagi, itu membuat mereka dengan file baru disertakan.
