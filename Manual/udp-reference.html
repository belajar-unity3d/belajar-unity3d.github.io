Referensi konsol UDP

Bagian ini memberikan ikhtisar tingkat tinggi dari arsitektur informasi konsol UDP.

Konsol UDP mengandung bilah navigasi yang memungkinkan Anda menavigasi antara bagian berikut dari konsol UDP:

Overview

Tab Ikhtisar konsol UDP mengandung informasi yang berguna tentang cara menggunakan konsol UDP, termasuk:

  • Video tutorial
  • Panduan walkthrough untuk membantu Anda melacak kemajuan Anda
  • Informasi khusus toko
  • FAQ

Permainan saya

Tab Game Saya menampilkan proyek UDP Anda. Dari sini Anda dapat beralih antara proyek UDP, menciptakan permainan baru, dan game archive dan restore.

Anda dapat menyaring daftar permainan dengan status, misalnya:

  • All Games
  • Published to UDP
  • Submitted to stores
  • Live on stores

Jika Anda memilih Submitted to stores atau Live on stores, Anda juga dapat menyaring daftar permainan dengan menyimpan.

Ketika Anda memilih permainan, tab berikut tersedia:

  • Info Game
  • Publish
  • Status

Setiap kartu permainan menampilkan status permainan. Status Diterbitkan menunjukkan permainan telah menyelesaikan langkah-langkah yang tersedia untuk dikemas ulang dan diserahkan ke toko.

Aplikasi Signature

UDP menggunakan kunci pribadi yang ditugaskan untuk menandatangani file APK yang dikemas ulang yang dikirimkan ke toko.

Unity merekomendasikan bahwa Anda memilih Export and upload the key and certificate dan menggunakan aplikasi Anda sendiri menandatangani kunci pribadi. Jika permainan Anda dirilis di Google Play, gunakan kunci yang sama seperti di Google Play. Ini secara signifikan mengurangi kemungkinan Google Play Protect menandai permainan Anda ketika pengguna menginstalnya.

Note: Jika permainan Anda ditandai, Anda dapat menarik ke Google here.

Tabel di bawah ini menggambarkan bagian Tanda Tangan dari tab Game Info.

Property Description
Export and upload the key and certificate (recommended) UDP menggunakan kunci pribadi penandaan Aplikasi yang Anda unggah untuk menandatangani build yang dipaketkan. Jika Anda memilih opsi ini, itu berlaku untuk semua toko yang Anda kirimkan permainan Anda.
Let UDP create and manage App signing private key UDP menghasilkan kunci pribadi penandaan aplikasi yang berbeda untuk setiap toko permainan dikemas ulang. Login Pilihan ini membuat permainan Anda lebih rentan terhadap peringatan Google Play Protect.
This option leaves your game more vulnerable to Google Play Protect warnings.

Note: Peringkat tanda tangan aplikasi hanya didukung untuk Versi SDK Android 18 dan di atas. Jika permainan Anda menggunakan versi SDK yang lebih rendah daripada ini, gunakan tanda tangan algoritma RSA atau meningkatkan tingkat API Minimum permainan.

Mengubah kunci

Anda dapat beralih dari menggunakan tombol UDP ke kunci Anda sendiri kapan saja. Perubahan ini hanya berlaku untuk toko-toko yang belum dikemas ulang untuk (dan oleh karena itu ditandatangani). Jika Anda sebelumnya telah mengirim ulang untuk toko menggunakan tombol UDP, toko itu akan selalu menggunakan tombol UDP.

Anda hanya dapat beralih dari menggunakan kunci Anda sendiri ke tombol UDP jika Anda belum membongkar permainan Anda (dan kunci Anda sendiri) untuk toko apa pun.

Kunci yang dihasilkan oleh UDP

UDP menggunakan informasi perusahaan untuk menghasilkan keystore untuk file APK repacked ketika kondisi berikut dipenuhi:

  • File APK yang diunggah oleh pengembang tidak mengandung file keystore.
  • Permainan belum dikemas dalam toko mitra.

Jika kondisi ini tidak terpenuhi, UDP menggunakan keystore UDP yang ada jika tersedia, atau menggunakan informasi pengembang untuk menghasilkan file tanda tangan baru.

Detail

Jika pengajuan Anda hilang apa pun, konsol UDP menampilkan kesalahan atau pesan peringatan. Pilih Detail untuk memperluas informasi lebih lanjut tentang masalah.

Kesalahan ditampilkan dalam panel merah. Anda harus memperbaiki kesalahan sebelum Anda dapat mengirimkan permainan Anda. Pilih Modify untuk pergi ke daerah erron untuk memperbaiki masalah.

Anda juga dapat memilih untuk hanya mengirimkan ke toko di mana tidak ada kesalahan.

Peringatan ditampilkan dalam panel kuning. Anda dapat memberhentikan peringatan yang Anda putuskan untuk mengabaikan.

Game Anda dapat memiliki status berikut:

  • Repacked: permainan Anda berhasil dikemas dengan SDK toko yang dipilih
  • Published: permainan Anda telah berlalu semua langkah yang diperlukan untuk toko ketika repacking dan menyerahkan ke toko itu
  • Pending: permainan Anda sedang dikemas dengan SDK toko yang dipilih
  • Failed: permainan Anda tidak bisa dibongkar dengan SDK toko yang dipilih
  • Canceled: pengajuan permainan Anda dibatalkan oleh Anda atau seseorang dari Organisasi Anda

Daftar Toko

Toko Sign-Ups tab menampilkan informasi tentang toko-toko Anda dapat mendistribusikan permainan Anda untuk menggunakan UDP.

Pilih tautan lebih untuk informasi lebih lanjut di toko, seperti gambaran, FAQ, dan tautan berguna lainnya yang spesifik untuk setiap toko.

Untuk gambaran informasi untuk semua toko, pilih Compare stores untuk membuka Partner Stores cheatsheet. Ini membandingkan rincian untuk semua toko, termasuk data seperti jumlah pengguna, negara di mana toko tersedia, dan apakah atau tidak toko mendukung game premium.

WordPress.org

Tab Pelaporan menampilkan informasi kinerja untuk permainan Anda yang diterbitkan. Pantau kinerja permainan yang dipublikasikan dari Das Pelaporan, yang dapat diakses dari bilah navigasi atas.

Anda dapat menyaring informasi yang disajikan di dashboard Pelaporan dengan permainan dan toko.

Jika Anda tidak memilih apa pun di filter All Games dan All Stores untuk seluruh dashboard Pelaporan, Anda dapat melihat kerusakan pada tingkat grafik individu (di mana berlaku). Pada grafik yang didukung, pilih filter by store atau by game untuk melihat kinerja di toko tertentu atau untuk permainan individu.

Dasbor Pelaporan dibagi menjadi lima panel, semua tunduk pada permainan dan Toko filter. Untuk panel yang tunduk pada filter tanggal, Anda dapat mengekspor data grafik sebagai file CSV dan gambar. Untuk mengunduh data sebagai file CSV atau gambar, pilih ikon yang sesuai untuk grafik.

Note: The Huawei AppGallery hanya mengembalikan informasi pendapatan ke UDP jika Anda telah mengatur validasi sisi server menggunakan QueryOrder. Jika Anda menggunakan notifikasi callback, tidak ada grafik atau informasi terkait pendapatan yang tersedia untuk Huawei.

Nilai data untuk file CSV

Setiap baris dalam file CSV dari Pembelian In-App dapat berisi nilai-nilai berikut. Jangan lewatkan nilai kosong.

Field Description
product_id Mewakili ID produk IAP yang unik. produk_id perlu memulai dengan huruf yang lebih rendah atau digit dan harus terdiri dari hanya huruf huruf huruf kecil (a-z), digit (0–9), underscores (_), dan periode (.)
language; product_name; description Saat mengatur nilai bahasa, gunakan kode bahasa yang tercantum dalam TABLE A.
UDP menggunakan en-US sebagai bahasa default.
Gunakan semicolon (;) untuk bahasa terpisah, nama dan deskripsi produk. Login Untuk menyertakan versi lokal dari nama produk item dan deskripsi, daftar bahasa default, nama produk_, dan deskripsi, diikuti oleh bahasa, nama produk_, dan deskripsi untuk setiap penduduk setempat. Dalam contoh berikut, produk telah en-US (ENGLISH) sebagai bahasa default dan zh-CN (CHINESE) sebagai lokalisasi:
en-US; Produk 1; Ini adalah produk pertama saya; zh-CN; Brexit1; Riwayat situs web1.
Catatan: UDP hanya mengimpor bidang untuk bahasa yang telah didefinisikan dalam bagian

consumable Set ke TRUE atau FALSE (karena kasus) untuk menentukan apakah atau bukan produk IAP dapat dikonsumsi.
auto_convert_prices Jika benar, UDP secara otomatis mengubah harga ke mata uang lain. Untuk mengecualikan mata uang dari konversi, tentukan mata uang dan harganya di bidang currency;price.
