Windows Player: IL2CPP membangun file

Proyek menggunakan IL2CPPBack-end scripting bersatu yang dapat Anda gunakan sebagai alternatif untuk Mono ketika proyek bangunan untuk beberapa platform. More info
Lihat di Glossary scripting backendKerangka kerja yang skrip di Unity. Unity mendukung tiga backend scripting yang berbeda tergantung pada platform target: Mono, .NET dan IL2CPP. Platform Windows Universal, namun hanya mendukung dua: .NET dan IL2CPP. More info
Lihat di Glossary biasanya (dapat berbeda karena membuat pengaturan) menghasilkan file ini:

File berikut umum untuk Proyek menggunakan Mono atau IL2CPP:

File: Description:
a_Data Folder dengan data permainan.
a.exe Game utama dapat dieksekusi.
UnityCrashHandler64.exe Kecelakaan handler executable.
UnityPlayer.dll Perpustakaan Unity Player berisi semua kode asli.
WinPixEventRuntime.dll PIX untuk Windows runtime. File ini hanya akan hadir dalam development buildsMembangun pengembangan termasuk simbol debug dan memungkinkan Profiler. More info
Lihat di Glossary.

File berikut umum untuk Proyek menggunakan Mono atau IL2CPP:

File: Description:
a_BackUpThisFolder_ButDontShipItWithYourGame Folder yang berisi data yang diperlukan untuk debug permainan Anda, termasuk file PDB (debug info) dan kode C++ yang dihasilkan dari scriptsSepotong kode yang memungkinkan Anda untuk membuat Komponen Anda sendiri, memicu peristiwa permainan, memodifikasi sifat komponen dari waktu ke waktu dan menanggapi input pengguna dengan cara apa pun yang Anda sukai. More info
Lihat di Glossary Anda. Anda harus kembali folder ini untuk setiap membangun kapal, tetapi tidak mendistribusikannya.
GameAssembly.dll Perpustakaan yang berisi runtime IL2CPP dan semua kode skrip Anda.
SymbolMap File yang berisi daftar semua alamat fungsi yang dikelola dan panjangnya. IL2CPP perlu ini untuk menyelesaikan jejak stack yang dikelola. Jika Anda menghapusnya, permainan Anda masih akan berjalan tetapi pengecualian tidak akan menghasilkan tumpukan panggilan yang masuk akal
