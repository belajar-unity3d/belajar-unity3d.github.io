Filter searches

Penyaringan mempersempit ruang lingkup pencarian Anda ke penyedia tertentu. Anda dapat menyaring pencarian dengan cara berikut:

  • Menyiapkan filter pencarian terus-menerus untuk mengontrol penyedia yang Cari menggunakan untuk pencarian biasa.

  • Gunakan token pencarian Penyedia Pencarian di bidang pencarian hanya menampilkan hasil dari penyedia tersebut.

  • Batasi hasil pencarian Anda dengan menggunakan sub-filter dan operator query dan menggunakan keywords yang tersedia untuk indeks Anda.

  • Lihat daftar filter pencarian tambahan here.

Persistent search filters

Anda dapat beralih sementara Penyedia Pencarian di dan dari pane Filter. Ini dapat membantu mengurangi jumlah item yang dicari kembali, yang nyaman jika Anda sudah tahu jenis item apa yang Anda cari. Penyedia yang beralih pada setiap waktu tertentu adalah Penyedia Pencarian Aktif.


Pencarian Penyedia menu drop-down

Ketika Anda beralih penyedia pencarian di menu Penyedia Pencarian, Cari "menutup" mereka daripada menonaktifkan mereka sepenuhnya. Penyedia Pencarian yang melakukan pengindeksan latar belakang atau operasi tersembunyi lainnya terus melakukannya ketika berlumpur.

Sitemap Untuk penyedia pencarian toggle permanen di dan mati, mengubahnya di Note:.Search preferences.

Untuk mengatur filter pencarian terus-menerus:

  1. Pilih Edit > Search All untuk meluncurkan Pencarian, dan di area Penyedia Pencarian pilih More Options (:)
  2. Aktifkan atau menonaktifkan Penyedia Pencarian yang ingin Anda masukkan/tidak termasuk dari pencarian berikutnya.

Sitemap Anda dapat menggunakan Tip: (up panah) dan ↑ (down panah) untuk siklus melalui filter yang tersedia, dan ↓ toggle filter.Space to toggle a filter.

Search tokens

Setiap Penyedia Pencarian memiliki string teks unik yang disebut token pencarian, juga disebut ID filter. Ketika Anda menetapkan permintaan pencarian dengan token pencarian penyedia, Cari membatasi ruang lingkup pencarian untuk penyedia itu.

Sebagai contoh, p: adalah token pencarian untuk Penyedia Pencarian Aset. Ketika Anda memasukkan p:Player di bidang pencarian, Cari pencarian untuk Aset yang sesuai dengan istilah "Player" (misalnya, aset dengan "Player" dalam nama mereka).

Lihat Penyedia Pencarian untuk daftar token pencarian untuk Penyedia Pencarian.

Lihat Token pencarian tambahan untuk daftar token pencarian untuk PrefabsJenis aset yang memungkinkan Anda untuk menyimpan GameObject lengkap dengan komponen dan properti. Prefab bertindak sebagai template dari mana Anda dapat membuat instance objek baru di tempat kejadian. More info
Lihat di Glossary, File, Jenis, Properti, dan Pencarian Dependencies.

Menggabungkan token pencarian

Anda dapat menggabungkan token pencarian untuk membuat pertanyaan yang lebih kompleks.

  • Pertanyaan ditulis pada satu baris dengan satu ruang karakter antara token.
  • Ruang karakter antara setiap token baru adalah operasi "Dan", sehingga kedua filter harus benar untuk permintaan untuk mengembalikan hasilnya. Tambahkan operator lain (atau, <,>) untuk mengembalikan hasil yang berbeda.
  • Jika seorang pencari penyedia filter token (h:, p:) digunakan, itu harus menjadi komponen pertama dalam kueri.

Berikut adalah beberapa contoh:

Query Description
h: t:meshrenderer p(castshadows)!="Off" Cari semua jala statis dalam SceneAdegan berisi lingkungan dan menu permainan Anda. Pikirkan setiap file Adegan unik sebagai tingkat yang unik. Di setiap Adegan, Anda menempatkan lingkungan, hambatan, dan dekorasi, pada dasarnya merancang dan membangun permainan Anda dalam potongan-potongan. More info
Lihat di Glossary yang melemparkan bayangan.
h: t:light p(color)=#FFFFFF p(intensity)>7.4 Cari semua lampu di Adegan dengan warna tertentu dengan kecerahan lebih tinggi dari 7.4.
h: path:/Collectables t:collectable Temukan semua objek dengan komponen Collectable yang terletak di jalur /Collectables.

Search expressions

Cari ekspresi memungkinkan Anda untuk menambahkan ke bahasa query pencarian untuk mengekspresikan pertanyaan kompleks yang referensi beberapa penyedia, misalnya, untuk mencari semua objek dalam adegan yang menggunakan shaderProgram yang berjalan di GPU. More info
Lihat di Glossary yang tidak dikompilasi. Lihat Search expressions untuk informasi lebih lanjut.
