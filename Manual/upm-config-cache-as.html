Sesuaikan lokasi cache Aset Store

Manajer Paket mempertahankan cache untuk paket yang Anda dapatkan dari Asset StoreSebuah perpustakaan yang berkembang dari aset gratis dan komersial yang diciptakan oleh Unity dan anggota komunitas. Menawarkan berbagai aset, dari tekstur, model dan animasi ke seluruh contoh proyek, tutorial dan ekstensi Editor. More info
Lihat di Glossary. Paket Manager menyimpan cache ini di lokasi default, yang dapat Anda tumpangkan.

Anda mungkin ingin menimpa lokasi cache Asset Store untuk beberapa alasan:

  • Anda ingin menghemat ruang di drive internal Anda.
  • Anda ingin menyimpan cache di drive bersama, yang dapat diakses oleh orang lain.
  • Anda ingin menyimpan cache di folder yang tidak dalam direktori sistem.

Secara default, Manajer Paket menggunakan struktur folder berikut untuk cache paket Aset Store:

<asset-store-cache-root>
  └── Asset Store-5.x
      └── <subfolders for Asset Store vendors>

Untuk informasi di lokasi default dari akar cache paket Asset Store, lihat Lokasi file aset yang diunduh.

Note: Manajer Paket mempertahankan cache terpisah untuk paket registry dan paket Asset Store. Untuk informasi tentang mempersembahkan lokasi default untuk cache global, lihat Sesuaikan lokasi cache global.

Untuk menimpa lokasi default dari cache Paket Manager untuk paket Aset Store, Anda dapat menggunakan metode berikut:

  • Jendela Preferences adalah metode yang disarankan karena kemudahan dan keabadiannya. Ketika Anda mengubah lokasi paket cache menggunakan metode ini, Unity menyimpan preferensi untuk sesi saat ini dan sesi masa depan.
  • Metode ASSETSTORE_CACHE_PATH tidak permanen, tetapi pengguna canggih mungkin menemukannya lebih disukai dalam kasus penggunaan tertentu.

Catatan penting sebelum Anda mulai

  • Terlepas dari metode yang Anda pilih, mengubah lokasi cache memberitahu Manajer Paket untuk menggunakan lokasi itu akan maju. Paket yang ada yang disimpan dalam lokasi asli tetap berada di struktur folder tersebut.
  • Jika Anda menggunakan metode variabel lingkungan untuk mengubah lokasi cache, maka Anda tidak dapat menggunakan jendela Preferensi untuk mengubah lokasi.

Menggunakan jendela Preferensi

Untuk menggunakan jendela Preferensi untuk menimpa lokasi default dari cache paket Asset Store, ikuti langkah-langkah ini.

  1. Gunakan salah satu metode berikut untuk membuka jendela Preferensi:

    • Buka menu Unity (macOS) atau menu Edit (Windows dan Linux) dan pilih Preferences.
    • Buka jendela Paket Manager, klik ikon pengaturan , dan pilih Preferences.

  2. Pilih kategori Package Manager.

  3. Di bawah My Assets membuka menu di samping Cache Location .

  4. Pilih Change Location.

  5. Pilih lokasi baru untuk cache paket Asset Store.

Menggunakan variabel lingkungan

Dalam skenario yang melibatkan otomasi atau integrasi berkelanjutan, kurang praktis dan lebih rentan kesalahan untuk mengkonfigurasi pengaturan dalam file konfigurasi atau jendela preferensi. Dalam skenario seperti itu, Anda mungkin mempertimbangkan pengaturan variabel lingkungan ASSETSTORE_CACHE_PATH untuk menimpa lokasi default cache paket Asset Store.

Important: Ikuti panduan ini ketika Anda menggunakan metode ini:

  • Tutup Editor Unity dan Hub Unity jika sudah berjalan sebelum mengatur variabel lingkungan.
  • Luncurkan Editor Unity atau Hub Unity dari perintah yang sama atau sesi terminal di mana Anda mengatur variabel lingkungan.
  • Anda harus mengatur variabel lingkungan ASSETSTORE_CACHE_PATH setiap kali Anda meluncurkan Unity.

Untuk informasi tentang mengatur variabel lingkungan, merujuk pada dokumentasi untuk sistem operasi Anda. Untuk pengenalan variabel lingkungan, lihat url https://en.wikipedia.org/wiki/Environment_variable.
