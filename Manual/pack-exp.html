Paket eksperimental

Paket eksperimental adalah paket baru atau modifikasi eksperimental yang dibuat untuk paket matang. Unity tidak mendukung paket eksperimental karena mereka berada di tahap awal perkembangan.

Sitemap Sebelum versi Editor Unity 2021.1, Manajer Paket menggunakan state "Preview" untuk menggambarkan paket yang bereksperimen atau berisiko tetapi tidak matang. The Package Manager menggunakan state “Preview” untuk menggambarkan paket yang belum sepenuhnya divalidasi sebagai aman untuk digunakan dalam produksi. Mulai dengan 2021.1, negara “Preview” tidak ada lagi, dan paket bisa menjadi “Experimental” atau “Pre-release.” Ini memberikan perbedaan yang lebih jelas antara paket yang matang tetapi berisiko untuk digunakan, dan paket yang hampir sepenuhnya matang.Note: Before Unity Editor version 2021.1, the Package Manager used the “Preview” state to describe packages that are experimental or risky but otherwise mature. The Package Manager used the “Preview” state to describe packages that had not yet been fully validated as safe to use in production. Starting with 2021.1, the “Preview” state no longer exists, and packages can either be “Experimental” or “Pre-release.” This provides a clearer distinction between packages that are mature but risky to use, and packages that are almost fully mature.

Paket eksperimental dapat melalui banyak perubahan sebelum mereka siap untuk rilis dalam versi khusus Unity. Pada beberapa titik di masa depan, mereka mungkin melewati persyaratan verifikasi tetapi mereka mungkin juga menjadi perbuatan. Karena tidak ada jaminan dukungan masa depan, Anda tidak boleh menggunakan paket eksperimental dalam produksi.

Paket dalam keadaan eksperimental tidak biasanya muncul dalam konteks Unity Registry dari Manajer Paket, meskipun mereka berada di server registry paket resmi Unity. Paket ini tidak dapat ditemukan di jendela Paket Manager karena:

  • Mereka terlalu berisiko untuk digunakan dalam produksi. Beberapa paket ini membutuhkan banyak pelatihan dan keahlian dan disarankan hanya dalam keadaan tertentu.
  • Mereka menyediakan fungsi bersama atau tambahan untuk paket yang ada. Anda tidak boleh menggunakannya sendiri karena paket "dukungan".

Paket eksperimental yang tidak dapat ditemukan masih dapat muncul di jendela Manajer Paket jika Anda telah menginstalnya di proyek Anda atau menginstalnya sebagai dependensi paket yang didukung. Namun, mereka tersembunyi sehingga Anda tidak menemukan mereka dengan kecelakaan dan menggunakannya tanpa menyadari risiko. Jika mereka muncul di Editor, mereka selalu ditandai dengan label , dan menu berikut muncul sebagai peringatan di Editor:

Anda dapat membuka menu Experimental Packages In Use dan pilih Dismiss jika Anda tidak ingin melihat peringatan ini untuk proyek ini. Anda juga dapat membuka menu dan pilih Show Experimental Packages untuk membuka Manajer Paket dengan daftar paket eksperimental di proyek Anda.

Untuk daftar paket stabil yang diverifikasi untuk rilis ini, lihat Promo.

Untuk informasi lebih lanjut tentang status paket, lihat Paket negara dan siklus hidup.
