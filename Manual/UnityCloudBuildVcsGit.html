Menggunakan Dashboard Pengembang Unity untuk mengkonfigurasi Build Cloud untuk Git

Cloud Build mendukung proyek yang disimpan dalam repositori Git. Repositori Anda dapat dihosting pada GitHub, GitLab, Bitbucket, atau server pribadi.

Penutup topik ini:

  • Mengkonfigurasi GIT di Dasbor

  • Menggunakan repositories pribadi

  • Menyiapkan platform membangun target

  • Menggunakan modul Git

Mengkonfigurasi Git di Dasbor

Untuk mengkonfigurasi Cloud Membangun Proyek Anda dari repositori GitHub:

  1. Masuk ke .
  2. Pada halaman ProjectsDalam Unity, Anda menggunakan proyek untuk merancang dan mengembangkan permainan. Proyek menyimpan semua file yang terkait dengan permainan, seperti file aset dan Adegan. More info
    Lihat di Glossary    , pilih Proyek Anda.
  3. DevOps > Cloud Build > Build History.
  4. Di jendela Build History, klik Set up Cloud Build.

Anda dapat mengkonfigurasi akses ke repositori Anda menggunakan protokol otorisasi berikut:

  • OAuth (default)
  • SSH

Untuk mengkonfigurasi Cloud Build untuk menggunakan OAuth untuk mengakses repositori Anda:

  1. Di jendela Source Control, klik GitHub, Bitbucket, atau ubin GitLab.
  2. Menerapkan akses ke akun Anda sesuai dengan layanan kontrol kode sumber yang dipilih.
  3. Di jendela Select a Repository, pilih repositori dari mana untuk membangun Proyek dan kemudian klik tombol NEXT: TARGET SETUP.

Note: Cloud Build menggunakan kredensial Anda saat ini untuk mengambil daftar repositori jika Anda saat ini masuk ke layanan manajemen kode sumber. Untuk mengkonfigurasi Proyek yang mengambil repositori dari akun yang berbeda, menandatangani layanan manajemen kode sumber pertama, kemudian mengkonfigurasi Proyek.

Untuk langkah berikutnya, lihat Menyiapkan platform membangun target.

Untuk mengkonfigurasi Cloud Build untuk menggunakan SSH untuk mengakses repositori Anda:

  1. Di jendela Source Control, klik tab Manual.

  2. Dalam bidang SCM URL, masukkan URL server Git Anda dalam format berikut:

    • Non-SSL: host: pelabuhan
    • WordPress.org

  3. Untuk menghubungkan ke repositori Anda, Anda harus menentukan URL ke server Git Anda. Untuk informasi tentang format URL, lihat URL syntax. Sitemap Jika Anda hosting Git di server pribadi, Anda harus menggunakan SSH untuk terhubung ke repositori Anda.Note: If you are hosting Git on a private server, you must use SSH to connect to your repository.

  4. Dari menu drop-down SCM Type, pilih GIT.

  5. Klik tombol NEXT: ACCESS.

Ketika Cloud Build terhubung ke situs hosting, secara otomatis mendeteksi apakah repositori Anda adalah publik atau pribadi. Jika repositori Anda adalah publik, Cloud Build secara otomatis terhubung ke dalamnya, dan Anda dapat melewatkan ke Menyiapkan platform membangun target. Jika repositori Anda pribadi, lihat bagian Menggunakan repositories pribadi.

URL

Untuk menghubungkan ke repositori Anda, Anda harus menentukan URL ke server Git Anda. Anda dapat menentukan URL untuk menggunakan protokol https://.

Note: Login Jika Anda terhubung melalui SSH, ada langkah keamanan baru. Lihat Meningkatkan keamanan protokol Git di GitHub untuk informasi lebih lanjut.

Berikut ini adalah contoh URL untuk GitHub, bitbucket, dan GitLab:

  • url https://github.com/youraccount/yourrepo
  • git@bitbucket.org
  • git@gitlab.com

Note: Jika Anda hosting Git di server pribadi, Anda harus menggunakan SSH untuk terhubung ke repositori Anda.

Unity Cloud Build secara otomatis menulis ulang URL ke format itu perlu.

Menggunakan repositories pribadi

Jika repositori Anda pribadi, Build Cloud harus menggunakan SSH untuk terhubung kenya. Ketika Cloud Build mendeteksi bahwa repositori adalah pribadi, itu menampilkan layar Grant us access to your source control.

Menyiapkan kunci SSH Anda di GitHub

Untuk menambahkan kunci SSH Anda ke proyek Anda di GitHub:

  1. Masuk ke GitHub.

  2. Di sudut kanan atas halaman GitHub, klik foto profil Anda, lalu klik profil Anda.

  3. Pada halaman profil Anda, klik Repositories, lalu klik nama repositori Anda.

  4. Di repositori Anda, klik tab Settings.

  5. Di sidebar, klik Deploy Keys, lalu klik tombol Add deploy key.

  6. Dalam kotak teks Title, nama tipe untuk mengidentifikasi kunci ini.

  7. Di bidang Key, tempel di kunci publik Anda dari jendela Grant us access to your source control, dan kemudian klik Add key button.

Untuk informasi lebih lanjut tentang menggunakan kunci, lihat Mengelola Kunci Deploy dalam dokumentasi GitHub.

Anda juga dapat menambahkan kunci SSH ke akun GitHub Anda sehingga semua repositories di akun Anda dapat diakses oleh Cloud Build. Untuk informasi lebih lanjut, lihat Menambahkan kunci SSH baru ke akun GitHub Anda dalam dokumentasi GitHub.

Menyiapkan kunci SSH Anda di Bitbucket

Untuk menambahkan kunci SSH Anda ke akun Bitbucket Anda:

  1. Login ke Bitbucket.

  2. Klik avatarAntarmuka untuk penargetan ulang animasi dari satu rig ke yang lain. More info
    Lihat di Glossary     Anda di sebelah kiri bawah halaman.

  3. Klik Bitbucket settings.

  4. Pada bagian Settings page, di bagian SECURITY, klik SSH Keys.

  5. Pada SSH Keys, klik tombol Add key.

  6. Dalam bidang Label, masukkan nama yang dapat dikenal untuk kunci (seperti Unity Cloud BuildLayanan integrasi berkelanjutan untuk proyek Unity yang mengotomatiskan proses pembuatan build pada server Unity. More info
    Lihat di Glossary    .)

  7. Tempel kunci Unity Cloud Build SSH dari jendela Grant us access to your source control ke bidang Key.

  8. Klik Add Key.

Pada jendela Grant us access to your source control, klik Next: Target Setup.

Menyiapkan platform membangun target

Di dasbor, di jendela NEW BUILD TARGET: BASIC INFO:

  1. Dalam bidang Target Label, masukkan nama untuk membangun.

  2. Jika folder akar repositori Anda tidak mengandung Aset dan Project settingsPengumpulan yang luas dari pengaturan yang memungkinkan Anda untuk mengkonfigurasi bagaimana Fisika, Audio, Jaringan, Grafik, Input dan banyak bidang lain dari jangkauan proyek Anda. More info
    Lihat di Glossary     Anda, di bidang Project Subfolder, masukkan URL folder yang berisi Aset dan Pengaturan Proyek Anda.

  3. Dari menu drop-down Unity Version, pilih versi Unity yang harus membangun Proyek.

  4. Jika Anda tidak ingin Proyek untuk secara otomatis membangun setiap repositori Anda diperbarui, klik toggle Auto-build untuk menonaktifkan fitur ini.

Jika Anda membangun platform iOS atau Android, langkah berikutnya Anda adalah memasukkan kredensial untuk membangun. Klik Next: Credentials. Untuk semua platform lain, klik Next: Build untuk menyelesaikan konfigurasi dan memulai build awal.

Android

  • Sebuah Bundle ID untuk mengidentifikasi aplikasi Anda secara unik di perangkat dan di Google Play Store.

  • Masukkan kredensial Android keystoreSistem Android yang memungkinkan Anda menyimpan entri kunci kriptografi untuk keamanan perangkat yang ditingkatkan. More info
    Lihat di Glossary     atau pilih Auto Generated Debug Keystore untuk menggunakan keystore pengembangan. Untuk informasi lebih lanjut tentang keystore Android, lihat Sistem Keystore Android.

Di jendela iOS SIGNING CREDS menyediakan informasi berikut:

  • A Bundle ID untuk mengidentifikasi aplikasi Anda secara unik di perangkat.

  • Versi Xcode yang untuk membangun aplikasi.

  • Masukkan kredensial IOS Anda. Untuk informasi lebih lanjut tentang kredensial iOS, lihat API.

Menggunakan modul Git

Jika proyek Anda menggunakan pribadi Submodul Git, pastikan bahwa URL yang ada dalam file .gitmodules Anda menggunakan sintaks git@ bukan https://.

Contoh:

  • git@github.com:youraccount/yourrepo.git (untuk GitHub)

  • git@bitbucket.org

  • git@gitlab.com
