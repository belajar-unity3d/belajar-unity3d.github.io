Beralih ke versi paket lain

Informasi pada halaman ini hanya berlaku untuk paket yang diinstal dari Unity Registry, registry lingkup, atau dari Sumber lokal.

Note: Untuk memperbarui paket yang diinstal dari URL Git, Anda dapat menggunakan metode berikut:

  • Temukan paket di Manajer Paket, pilih, lalu klik tombol Update.
  • Menghapusnya sebagai ketergantungan Git menggunakan revisi baru. Untuk informasi lebih lanjut tentang cara menentukan revisi dengan Git dependenciesManajer Paket mengambil Ketergantungan git dari repositori Git langsung daripada dari registry paket. Ketergantungan Git menggunakan referensi URL Git bukan versi, dan tidak ada jaminan tentang kualitas paket, stabilitas, validitas, atau bahkan apakah versi yang tercantum dalam berkas package.json menghormati aturan Versi Semantic sehubungan dengan rilis resmi paket ini.   • Menghapusnya dari Registry Unity.

Anda dapat memperbarui paket sementara dalam konteks In Project, Unity Registry, atau My Registries:

  1. Buka jendela Paket Manager dan pilih In Project dari menu PaketPaket adalah koleksi aset untuk dibagikan dan digunakan kembali dalam Unity. Manajer Paket Unity (UPM) dapat menampilkan, menambahkan, dan menghapus paket dari proyek Anda. Paket-paket ini berasal dari Unity Package Manager dan memberikan metode mendasar untuk memberikan fungsi Unity. Namun, Unity Package Manager juga dapat menampilkan Paket Asset Store yang Anda unduh dari Asset Store. More info
    Lihat di Glossary    .

    Indikator panah muncul di sebelah paket apa pun yang tersedia.

  2. Pilih paket yang diinstal yang ingin Anda perbarui dari Daftar paket. Informasi paket muncul di details view.

  3. Ikon kunci () menunjukkan bahwa paket dan versi ini terkunci ke feature setSebuah fitur set adalah koleksi paket terkait yang dapat Anda gunakan untuk mencapai hasil tertentu dalam Editor Unity. Anda dapat mengelola set fitur langsung di Manajer Paket Unity. More info
    Lihat di Glossary     yang terpasang. Untuk membuka paket dan pilih versi lain, klik tombol Unlock.

    Note: Paket dibuka sementara. Jika Anda berubah menjadi list context yang berbeda, atau menutup Manajer Paket atau Editor, paket mengalihkan kembali ke keadaan terkunci lagi. Namun, jika Anda beralih versi ketika paket dibuka (misalnya, dengan metode Tambahkan paket berdasarkan nama), paket tetap dibuka.

  4. Pada Daftar paket, klik ikon expander ke kiri nama paket.

    Note: Jika Anda melihat tautan See other versions setelah memperluas paket, Anda dapat mengkliknya untuk melihat daftar semua versi yang tersedia untuk paket tersebut.

  5. Pilih versi paket saat ini untuk melihat rincian spesifik untuk versi itu di details view.

  6. Klik tombol Update to.

    Ketika progress bar selesai, fungsi baru apa pun segera tersedia.

Notes:

  • Jika Anda beralih ke versi paket yang lebih tua, Anda mungkin harus menjalankan API Updater pada konten paket.
  • Anda dapat beralih versi untuk beberapa paket dengan satu klik dengan menggunakan beberapa fitur pilih. Untuk informasi lebih lanjut, lihat Melakukan tindakan pada beberapa paket atau set fitur.
