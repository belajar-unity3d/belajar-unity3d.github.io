StyleSheets

Lembar Gaya dan pemilih AS dalam UI Toolkit digunakan untuk berbagi dan menerapkan gaya di berbagai elemen dan banyak Dokumen UI (UXML).

StyleSheets pane adalah di mana Anda bisa:

  1. Tambahkan, reorder, atau lepaskan Lembar Gaya (USS) ke Dokumen UI saat ini (UXML).
  2. Buat, reorder, menyalin, atau menghapus Selectors USS dalam StyleSheets untuk berbagi gaya di banyak elemen.

Hierarchy

Hierarchy pane mengandung tampilan pohon mewakili hierarki elemen dokumen saat ini. Ini adalah superset hirarki yang Anda lihat dalam representasi teks UXML dari Dokumen UI (UXML). Selain elemen yang dibuat langsung dari UXML, pane Hierarchy juga mengandung unsur-unsur yang dibuat secara dinamis dan hanya ada dalam runtime UI. Misalnya, pane Hierarchy mencakup unsur-unsur yang merupakan bagian dari instance Template (dokumen yang berbeda di dalam dokumen saat ini), di mana UXML hanya akan memiliki tag <Instance> tunggal.

Anda dapat menggunakan pane Hierarchy untuk:

  1. Pilih elemen untuk memeriksa atau mengedit.
  2. Potong, menyalin, menghapus, atau mengubah elemen.
  3. Buka dokumen template instanced sebagai sub-documents untuk pengeditan in-place (dalam konteks).
  4. Lihatlah beberapa daftar kelas gaya elemen, atribut nama, dan lampiran Lembar Gaya lokal.

Elemen muncul di pohon sesuai dengan atribut name mereka. Jika atribut name elemen tidak diatur, muncul sesuai dengan tipe C #nya. Anda dapat mengklik dua kali elemen untuk cepat mengubah nama itu.

Klik ikon â‹® di sudut kanan atas header pane Hierarchy untuk mengakses menu opsi tampilan.

  • Pilih Type untuk memaksa Hierarchy untuk menampilkan setiap elemen tipe C #, terlepas dari apakah memiliki nama.
  • Pilih Class List untuk menampilkan setiap daftar kelas gaya elemen di sebelah entri Hierarchy.
  • Pilih Attached StyleSheets untuk menampilkan Lembar Gaya yang melekat secara lokal di samping elemen mereka.

Navigasi di viewport

Pan dan zoom untuk menavigasi di viewport. Pembuat UI menyimpan setiap Dokumen UI (UXML) saat ini panci dan keadaan zoom, dan mengembalikannya setelah jendela UI Builder memuat ulang, reload domain, atau ketika Anda menutup dan membuka kembali Dokumen UI yang sama (UXML).

Ketika Anda membuat dan membuka dokumen baru, UI Builder mengatur ulang pan dan memperbesar negara.

Untuk panci, lakukan salah satu berikut:

  1. Klik dan tarik di Viewport.
  2. TahanCtrl/Cmd + Alt /Option dan klik dan tarik di Viewport.

Untuk memperbesar dan keluar, posisi pointer mouse di Viewport dan melakukan salah satu berikut:

  1. Posisikan pointer mouse di viewport, dan putar roda mouse.
  2. TahanAlt /Option dan drag yang tepat. Seret benar untuk memperbesar, dan meninggalkan untuk memperbesar.

Kode Pratinjau

Ketika Anda membangun UI, UI Builder secara otomatis menghasilkan teks UXML dan USS yang mendasarinya, dan menampilkannya dalam panci UXML Preview dan USS Preview .

Inspector

UI Builder Inspector pane mirip dengan Unity Inspector window. Inspector akan menampilkan sedikit konten yang berbeda, tergantung pada objek yang telah Anda pilih di dalam UI Builder:

  1. Elemen menampilkan bagian Attribute untuk atribut per-element (UXML), bagian StyleSheets untuk menetapkan kelas gaya dan melihat gaya yang diterapkan dari Lembar Gaya, dan bagian Inlined Styles yang memungkinkan pengeditan properti gaya per-element inline.
  2. Login Selector menampilkan bagian Style Selector untuk mengubah string query pemilih dan bagian Styles untuk mengedit properti gaya.
  3. Kanvas menampilkan bagian Canvas Size untuk mengubah ukuran Canvas, bagian Canvas Background untuk mengubah latar belakang kanvas edit-time, dan bagian Document Settings untuk pengaturan waktu edit per-document.
