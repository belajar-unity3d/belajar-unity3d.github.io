Menggunakan Dashboard Pengembang Unity untuk mengkonfigurasi Build Cloud untuk Mercurial

Cloud Build mendukung proyek yang disimpan dalam repositories Mercurial. Anda dapat menghosting repositori Anda pada Bitbucket atau server pribadi.

Untuk terhubung ke repositori Anda, Build Unity Cloud memerlukan bahwa Anda menyediakan kredensial akses yang tepat. Ini berlaku untuk repositori Mercurial yang dihosting di Bitbucket atau di tempat lain.

Mengkonfigurasi Mercurial di Dasbor

  1. Masuk ke .
  2. Dari halaman ProjectsDalam Unity, Anda menggunakan proyek untuk merancang dan mengembangkan permainan. Proyek menyimpan semua file yang terkait dengan permainan, seperti file aset dan Adegan. More info
    Lihat di Glossary    , pilih Proyek Anda.
  3. Dalam pane navigasi kiri, klik Cloud BuildLayanan integrasi berkelanjutan untuk proyek Unity yang mengotomatiskan proses pembuatan build pada server Unity. More info
    Lihat di Glossary     > Config.
  4. Jika ini adalah pertama kalinya Anda telah mengonfigurasi Build Cloud, Anda melihat halaman Membangun Sejarah memberi tahu Anda bahwa Cloud Build has not been set up for this project. Klik tautan Set up Cloud Build untuk melanjutkan.
  5. Di jendela Source Control, klik tab Manual.
  6. Dalam bidang SCM URL, masukkan URL server Mercurial Anda. Untuk informasi tentang format URL, lihat bagian URL syntax di bawah ini.
  7. Dari menu drop-down SCM Type, pilih Mercurial.
  8. Klik tombol Next Access.

URL

Untuk menghubungkan ke repositori Anda, Anda harus menentukan URL ke server Mercurial Anda. Anda dapat menentukan URL untuk menggunakan protokol berikut:

  • HTTPS

  • SSH

Berikut ini adalah contoh URL untuk bitbucket:

  • url https://github.com/youraccount/yourrepo

  • git@bitbucket.org

Gunakan format yang paling nyaman untuk Anda. Unity Cloud Build secara otomatis menulis ulang URL ke format itu perlu.

Menyiapkan kredensial Anda

Di server Mercurial Anda, buat nama pengguna untuk Build Unity Cloud dan menetapkan kata sandi yang aman. Jika host Mercurial Anda mendukungnya, buat akun pengguna yang membaca.

Pada jendela Grant us access to your source control, masukkan nama pengguna dan kata sandi, dan klik Next: Target Setup.

Menyiapkan platform membangun target

Di jendela Target Setup, pilih platform build.

Di Dasbor Pengembang, di jendela NEW BUILD TARGET: BASIC INFO:

  1. Dalam bidang Target Label, masukkan nama untuk membangun.

  2. Jika folder akar repositori Anda tidak mengandung Aset dan Project settingsPengumpulan yang luas dari pengaturan yang memungkinkan Anda untuk mengkonfigurasi bagaimana Fisika, Audio, Jaringan, Grafik, Input dan banyak bidang lain dari jangkauan proyek Anda. More info
    Lihat di Glossary     Anda, di bidang Project Subfolder, masukkan URL folder yang berisi Aset dan Pengaturan Proyek Anda.

  3. Dari menu drop-down Unity Version, pilih versi Unity yang harus membangun Proyek.

  4. Jika Anda tidak ingin Proyek untuk secara otomatis membangun setiap repositori Anda diperbarui, klik toggle Auto-build untuk menonaktifkan fitur ini.

Jika Anda membangun platform iOS atau Android, langkah berikutnya Anda adalah memasukkan kredensial untuk membangun. Klik Next: Credentials. Untuk semua platform lain, klik Next: Build untuk menyelesaikan konfigurasi dan memulai build awal.

Android

Pada jendela SIGNING CREDS menyediakan informasi berikut:

  • Sebuah Bundle ID untuk mengidentifikasi aplikasi Anda secara unik di perangkat dan di Google Play Store.

  • Masukkan kredensial Android keystoreSistem Android yang memungkinkan Anda menyimpan entri kunci kriptografi untuk keamanan perangkat yang ditingkatkan. More info
    Lihat di Glossary     atau pilih Auto Generated Debug Keystore untuk menggunakan keystore pengembangan. Untuk informasi lebih lanjut tentang keystore Android, lihat Sistem Keystore Android.
