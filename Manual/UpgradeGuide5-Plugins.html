Plugin di Unity 5.0

Catatan ini menyadari ketika proyek upgrade dari Unity 4 ke Unity 5, jika proyek Anda menggunakan plugin, termasuk plugin audio asli.

Plugin tidak lagi diperlukan untuk ditempatkan di Assets\Plugins <Platform> folder, mereka sekarang memiliki pengaturan untuk memeriksa platform yang kompatibel dan pengaturan spesifik platform (seperti pengaturan CPU kompatibel, SDK) dll. Secara default plugin baru ditandai sesuai dengan 'Any Platform'. Untuk membuat transisi lebih mudah dari versi Unity yang lebih tua, Unity akan memeriksa plugin folder apa yang terletak, dan mengatur pengaturan awal sesuai, untuk ex., jika plugin terletak di Assets\ Plugin\Editor, akan ditandai sebagai kompatibel dengan Editor hanya, jika plugin terletak di Assets\ Plugin\iOS, akan ditandai sesuai dengan iOS saja, dll. Juga silakan merujuk pada ‘PluginInspector dokumentasi’ untuk info lebih lanjut.

Plugin Asli dalam Editor

Plugin asli 32-bit tidak akan bekerja di editor 64-bit. Mencoba untuk memuat mereka akan mengakibatkan kesalahan yang masuk ke konsol dan pengecualian dibuang ketika mencoba memanggil fungsi dari plugin asli.

Untuk terus menggunakan plugin asli 32-bit di editor, gunakan versi editor 32-bit (dibuktikan sebagai installer terpisah).

Untuk memiliki versi 32-bit dan 64-bit dari plugin yang sama dalam proyek Anda, cukup letakkan di dua folder yang berbeda dan mengatur platform yang didukung tepat dalam pengaturan importir.

Pembatasan:

  • Saat ini kami tidak menyediakan editor 32-bit untuk OSX.

Plugin Audio Asli

Plugin dengan prefix "audioplugin" (case-insensitif) akan secara otomatis dimuat pada pemindaian sehingga mereka dapat digunakan dalam mixer audio. Hal yang sama berlaku untuk membangun mandiri di mana plugin ini dimuat pada waktu startup sebelum aset apapun belum dimuat, sebagai aset tertentu seperti mixer audio memerlukan mereka untuk dimuat untuk efek instan.
