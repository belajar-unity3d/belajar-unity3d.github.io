Lihat di Glossary. Remote Config adalah penawaran yang ditingkatkan yang memungkinkan Anda untuk mengubah perilaku dan penampilan permainan Anda tanpa memerlukan biner baru. Anda dapat mengaktifkan atau menonaktifkan fitur, mengubah spesifikasi permainan Anda untuk menargetkan audiens tertentu, atau menjalankan acara khusus dengan menjadwalkan rilis konten baru. Sitemap

Meskipun Anda dapat melihat kunci dan nilai-nilai kedua konfigurasi di Editor, Unity hanya memuat satu konfigurasi pada runtime. Unity menggunakan konfigurasi Pembangunan dalam mode Play dan dalam setiap build yang dibuat dengan kotak Development BuildMembangun pengembangan termasuk simbol debug dan memungkinkan Profiler. More info
Lihat di Glossary diperiksa pada Membangun jendela Pengaturan. Unity menggunakan konfigurasi Release dalam build non-development.

Sebelum menguji Pengaturan Jarak Jauh, Anda harus terlebih dahulu membuat pengaturan dan kemudian gunakan Komponen Pengaturan jarak jauh atau menulis kode yang diperlukan untuk memetakan pengaturan untuk variabel dalam permainan atau aplikasi Anda.

Untuk menguji dengan konfigurasi Development, klik tombol Play di Editor atau:

  1. Pergi ke File > Build Settings untuk membuka jendela Pengaturan Build.

  2. Periksa kotak Development Build.

  3. Klik tombol Build And Run.

Untuk menguji dengan konfigurasi Release:

  1. Pergi ke File > Build Settings untuk membuka jendela Pengaturan Build.

  2. Periksa kotak Development Build.

  3. Klik tombol Build And Run.

Menjelaskan Pengaturan Lokal

Untuk menguji apakah permainan Anda berperilaku seperti yang diharapkan pada lari pertama ketika tidak ada pengaturan lokal hadir, Anda dapat menghapus direktori Aplikasi.persistentDataPath. Jika Anda pengujian di Editor Unity, Anda harus keluar dari Editor sebelum menghapus direktori.
