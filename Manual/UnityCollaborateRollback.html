Mengembalikan versi proyek sebelumnya

Fungsi Restore dan Update memungkinkan Anda mengalihkan seluruh proyek Anda ke versi sebelumnya. Anda kemudian dapat mempublikasikan proyek yang diambil sebagai versi saat ini di cloud. Berkolaborasi memastikan bahwa semua versi proyek Anda utuh, bahkan setelah pengembalian, dan bahwa Anda dapat mengaksesnya setiap saat dalam siklus perkembangan.

Sitemap Ini menghapus semua perubahan yang tidak dipublikasikan secara permanen dan tidak dapat diubah.Note: This deletes all unpublished changes permanently and is irreversible.

  1. Pada Editor Unity toolbarSe baris tombol dan kontrol dasar di bagian atas Editor Unity yang memungkinkan Anda untuk berinteraksi dengan Editor dengan berbagai cara (misalnya scaling, terjemahan). More info
    Lihat di Glossary    , klik tombol kolaborasi.
  2. Klik History di panel Collaborate.

Panel History menampilkan daftar berkomitmen yang dilakukan pada proyek Anda.

Untuk menghapus semua perubahan lokal dan mengembalikan versi terbaru dari cloud, klik tombol perubahan pada berkomitmen terbaru dan klik Restore.

Untuk mengembalikan versi yang diterbitkan lebih awal dari versi saat ini di cloud, klik tombol perubahan pada komit dan klik Update.

Mengembalikan komit menggantikan semua file proyek lokal dengan file dari versi yang ditentukan. File sekarang terdaftar sebagai dimodifikasi di toolbar Collaborate. Biru plus sign (+) pada file atau folder menunjukkan bahwa itu dimodifikasi. Biru arrow (->) pada file atau folder berarti itu baru (lihat gambar di bawah).

Untuk mempublikasikan versi yang dipulihkan sebagai revisi saat ini di cloud, klik tombol Publish now!.
