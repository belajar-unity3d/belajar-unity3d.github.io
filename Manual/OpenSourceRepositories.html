Bagaimana berkontribusi pada Unity

Repositories sumber terbuka

Keputusan kami untuk memulai komponen open-sourcing Unity dimaksudkan untuk membantu kami terlibat dengan Anda, pelanggan dan pengguna kami. Kami ingin menyediakan semua sekuritas dan flexibilities yang memiliki akses sumber.

Kami selalu menyambut partisipasi kolaboratif dan pengembangan masyarakat. Harapan dan harapan kami bahwa dengan inisiatif ini, komunitas kami akan dapat memperluas Unity dengan cara yang sebelumnya tidak mungkin.

Repositories bersatu

Kami merekomendasikan bahwa Anda menggunakan Bitbucket ketika berkontribusi pada Unity, karena ini adalah apa yang kita cenderung gunakan secara internal, dan itu menjaga hal-hal sederhana. Unity memiliki sejumlah repositori open-source untuk membantu pengembang menggunakan perangkat lunak Unity:

http://bitbucket.org/Unity-Technologies/

Ini adalah repositori terbaru kami, digunakan terutama oleh tim UnityUI dan tim Konten R&D. Beberapa proyek inti R&D berakhir di sini (besar-besaran karena preferensi pengembang), seperti 5.3 Memory ProfilerJendela yang membantu Anda untuk mengoptimalkan permainan Anda. Ini menunjukkan berapa banyak waktu yang dihabiskan di berbagai bidang permainan Anda. Sebagai contoh, dapat melaporkan persentase waktu yang dihabiskan rendering, aimating, atau dalam logika permainan Anda. More info
Lihat di Glossary.

url https://github.com/Unity-Technologyes

Ini adalah repositori tertua kami, di mana beberapa proyek R & D inti kami menjaga pekerjaan open-source mereka. Proyek-proyek utama termasuk garpu Unity dari Mono (the C # runtime), dan Katana (sistem build/CI kami). Banyak teknologi yang digunakan untuk WebGLLogin API yang membuat grafis 2D dan 3D di browser web. Opsi build Unity WebGL memungkinkan Unity untuk mempublikasikan konten sebagai program JavaScript yang menggunakan teknologi HTML5 dan WebGL rendering API untuk menjalankan konten Unity di browser web. More info
Lihat di Glossary, seperti Emscripten, juga dihosting di sini.

URL https://github.com/unity3d-jp

Repositori ini digunakan oleh tim R&D dan Evangelism Unity Jepang. Ini berisi proyek demo, prototipe dan proyek samping dari karyawan kantor itu. Proyek yang menarik termasuk sistem Asset Graph yang akan datang untuk membangun AssetBundle, proyek demonstrasi yang menerangi cara menggunakan benang secara efektif dalam Unity, dan proyek contoh menunjukkan cara menggunakan iklan Unity.

Jika Anda ingin mempelajari lebih lanjut tentang berkontribusi pada pengembangan Unity, silakan lihat bagian pada Kontribusi Sumber Terbuka.
