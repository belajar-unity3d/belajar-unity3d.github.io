Verifikasi Penerimaan

Berikut adalah rincian implementasi:

Untuk iOS

receipt parameter

  • Jika Anda meninggalkan ini sebagai null, transaksi ini akan muncul di Pendapatan Tidak Diverifikasi
  • Jika Anda menulis plugin iOS asli
    • Untuk iOS 6 dan di bawah ini, kirim SKPaymentTransaction's transactionReceipt%
    • Untuk iOS 7 dan di atas, kirim base64 nilai yang dikodekan dari Penerimaan Aplikasi dari bundel aplikasi.
  • Jika Anda menggunakan plugin Unibill
    • Lulus dalam Login Barang "receipt" properti sebagai tanda terima.
  • Jika Anda menggunakan plugin Prime31
    • Lulus di properti "base64EncodedTransactionReceipt" StoreKitTransaction.

signature parameter

Lulus dalam null karena ini tidak digunakan.

receipt parameter

  • Jika Anda meninggalkan ini sebagai null, transaksi ini akan muncul sebagai Pendapatan yang Tidak Diverifikasi
  • Jika Anda menulis plugin Android Asli
    • Lulus pada data pembelian untuk pesanan, yang merupakan String dalam format JSON yang dipetakan ke INAPP_ Kunci PURCHASE_DATA dalam respons Intent.
  • Jika Anda menggunakan plugin Unibill
    • Parse Login Barang "receipt" properti sebagai JSON, dan lulus nilai properti "json".
  • Jika Anda menggunakan plugin Prime31
    • Facebook Twitter Google Plus Pinterest Email

signature parameter

  • Jika Anda meninggalkan ini sebagai null, transaksi ini akan muncul sebagai Pendapatan yang Tidak Diverifikasi
  • Jika Anda menulis plugin Android Asli
    • Lulus dalam signature, yang dipetakan ke kunci INAPP_DATA_SIGNATURE dalam respons Intent.
  • Jika Anda menggunakan plugin Unibill
    • Membuat Login Properti “receipt” barang sebagai JSON, dan lulus nilai properti tanda tangan.
  • Jika Anda menggunakan plugin Prime31
    • Pass di properti GooglePurchase “signature”.
