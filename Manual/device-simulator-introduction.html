Pengenalan Simulator perangkat

Simulator Perangkat adalah fitur Editor Unity yang mensimulasikan bagaimana aplikasi Anda muncul dan berperilaku pada perangkat seluler.

Simulator Perangkat terdiri dari:

  • Tampilan Simulator: Melihat aplikasi Anda pada perangkat seluler simulasi.
  • Kelas simulasi: Uji kode yang merespon perilaku spesifik perangkat.
  • Definisi perangkat: Jelaskan perangkat untuk mensimulasikan.
  • Plugin Simulator Perangkat: Konfigurasikan pandangan Simulator UI(User Interface) Memungkinkan pengguna untuk berinteraksi dengan aplikasi Anda. Unity saat ini mendukung tiga sistem UI. More info
    Lihat di Glossary    .

Kontrol dalam tampilan Simulator

Tampilan Simulator mensimulasikan banyak fitur umum perangkat seluler, termasuk:

  • Auto-rotasi
  • Area aman layar
  • Masukan sentuh

Pengaturan Pemain

Input sentuh simulasi

Jika Anda mengklik layar perangkat simulasi dengan kursor mouse, simulator perangkat menciptakan acara sentuh dalam solusi input aktif (baik Input ManagerPengaturan di mana Anda dapat mendefinisikan semua sumbu input yang berbeda, tombol dan kontrol untuk proyek Anda. More info
Lihat di Glossary, atau keduanya, tergantung pada pengaturan proyek Anda).

Note: Simulator Perangkat hanya mensimulasikan input ketika Editor berada dalam mode Bermain. Simulator Perangkat tidak mendukung multitouch; itu hanya dapat mensimulasikan satu sentuhan jari.

Limitations

Tujuan utama dari Simulator Perangkat adalah untuk melihat tata letak aplikasi Anda pada perangkat target dan menguji interaksi dasar. Ini tidak memberikan representasi yang akurat tentang bagaimana aplikasi Anda berjalan di perangkat.

Tampilan Simulator tidak mensimulasikan berikut:

  • Karakteristik kinerja perangkat, seperti kecepatan prosesor perangkat atau memori yang tersedia.
  • Kemampuan rendering perangkat.
  • Plugin asli yang tidak bekerja di Editor.
  • Platform #define directives untuk perangkat simulasi, seperti UNITY_IOS.
  • Rotasi giroskop.

Hanya satu tampilan Simulator dapat mensimulasikan satu kali. Ini adalah tampilan Simulator aktif.

  • Jika Anda hanya memiliki satu tampilan Simulator terbuka dan tidak ada pandangan Game terbuka, satu tampilan Simulator aktif terlepas dari apakah itu terlihat atau tidak.
  • Jika Anda memiliki beberapa tampilan Simulator terbuka dan tidak ada pandangan Game terbuka, tampilan Simulator terakhir yang fokus aktif.
  • Jika Anda memiliki campuran pandangan Simulator dan pandangan Game terbuka, jika Anda memfokuskan pandangan Game, Unity menonaktifkan semua simulator dan jika Anda fokus tampilan Simulator, tampilan Simulator tetap aktif saat memiliki fokus.

Simulator Perangkat tidak mensimulasikan semua API di kelas simulasi. Untuk informasi lebih lanjut, lihat Kelas simulasi.
