Integrasi Unity

Integrasi Unity memungkinkan Anda untuk menghubungkan Layanan Unity berikut dalam alur kerja pengembangan Anda ke alat non-Unity:

  • Build di CloudLayanan integrasi berkelanjutan untuk proyek Unity yang mengotomatiskan proses pembuatan build pada server Unity. More info
    Lihat di Glossary
  • Diagnostik Cloud
  • Collaborate

Untuk menambahkan Integrasi, Anda harus menjadi Pemilik atau Manajer di Proyek. Untuk informasi tentang menetapkan peran, lihat bagian Anggota & Grup dari Mengelola Organisasi Anda.

Integrasi Unity mendukung alat non-Unity berikut:

  • Webhooks: Callback yang ditentukan pengguna yang memungkinkan Unity untuk mengirim permintaan POST ke layanan eksternal.
  • Discord: Kirim notifikasi ke saluran Discord tim Anda melalui webhook yang ditentukan.
  • Slack: Kirim notifikasi ke saluran Slack tim Anda melalui webhook yang ditentukan.
  • Email: Kirim pemberitahuan kepada tim Anda melalui Email.
  • JIRA: Buat atau memperbarui tiket JIRA menggunakan webhook yang ditentukan JIRA.
  • Trello: Buat dan komentar pada kartu Trello.

Batas integrasi

Pengguna batas Pribadi Unity untuk satu Integrasi untuk semua acara, tetapi beberapa layanan Unity dapat menggunakan Integrasi itu,, dan pengguna dapat memiliki hingga 100 Integrasi. Jika tim Unity Anda, Pro, atau Plus langganan berakhir, Unity menyimpan semua Integrasi Anda saat ini, tetapi secara otomatis menonaktifkannya. Jika ini terjadi, Anda masih dapat masuk ke Dashboard Layanan Unity dan mengaktifkan satu Integrasi.

Menambahkan Integrasi

Untuk menambahkan Integrasi ke alur kerja Anda:

  1. Masuk ke .
  2. Pilih Proyek yang ingin Anda tambahkan Integrasi.
  3. Di bar navigasi kiri, pilih Settings > Integrations.
  4. Dasbor integrasi muncul. Klik tombol NEW INTEGRATION.

    %
  5. Pilih Integrasi yang ingin Anda aktifkan.
  6. Pilih acara yang Anda inginkan untuk memicu Integrasi, lalu klik tombol NEXT.
  7. Konfigurasikan opsi Integrasi. Pilihan ini bervariasi tergantung pada Integrasi yang dipilih, dan dijelaskan secara rinci di bagian di bawah ini.

Webhook

Untuk mengkonfigurasi Integrasi webhook, menyediakan informasi berikut:

Parameter Description
Nama tampilan Nama untuk mengidentifikasi Integrasi dalam daftar Integrasi.
Webhook URL URL dari endpoint server yang menerima permintaan Webhook POST dari layanan Unity.
Rahasia Otentikasi Rahasia klien dari aplikasi penerima Anda.
Jenis Konten Jenis MIME konten. Pilih jenis konten data Anda dari menu drop-down.
Verifikasi SSL / TLS Tick kotak centang ini untuk menonaktifkan verifikasi sertifikat keamanan SSL / TTLS. Sertifikat SSL / TTLS verifikasi
membantu memastikan bahwa data Anda dikirim dengan aman ke URL Webhook di atas. Mengubah opsi ini tidak dianjurkan, jadi hanya melakukan ini jika Anda benar-benar tahu apa yang Anda lakukan.

API Integration

Untuk mengkonfigurasi Integrasi Diskord, Unity memanggil aplikasi yang menggunakan API Discord untuk mendaftarkan webhook ke saluran Discord. Jika Anda tidak memiliki server Discord, lihat Bagaimana cara membuat server? dalam dokumentasi Diskord.

Important: Anda harus mengaktifkan webhooks di akun yang Anda mengintegrasikan.

Untuk menyelesaikan konfigurasi:

  1. Buat integrasi Discord baru dan pilih acaranya.
  2. Klik tombol NEXT. Masuk di layar login Discord.
  3. Layar dialog CONNECT TO DISCORD muncul.

    %
  4. Dari menu drop-down Select a server, pilih server Discord Anda.
  5. Dari menu Select a channel, pilih saluran untuk memposting pemberitahuan ke.
  6. Klik tombol Authorize.

API Integration

Untuk mengkonfigurasi Integrasi Slack, Unity memanggil aplikasi yang menggunakan Slack API untuk mendaftarkan webhook ke saluran Slack.

Setelah membuat integrasi Slack baru dan mengkonfigurasi peristiwa yang ingin Anda tampilkan, Anda harus memberi otorisasi Unity untuk mengakses server Slack Anda.

Jika Anda tidak memiliki server Slack, lihat Buat ruang kerja Slack dalam dokumentasi Slack.

Untuk menyelesaikan konfigurasi:

  1. Masuk ke akun Slack Anda.
  2. Di sudut kanan dari aplikasi, pilih ruang kerja Slack.
  3. Dari menu drop-down Post to, pilih saluran Slack untuk memposting pemberitahuan ke.
  4. Klik tombol Authorize.

API Integration

Integrasi Email memberitahukan anggota tim ketika acara tertentu untuk Proyek Anda terjadi.

  1. Buat integrasi email dari dashboard untuk melihat menu acara.
  2. Pilih peristiwa yang ingin Anda terima pemberitahuan email, kemudian klik tombol NEXT.

    Notehere di bagian bawah menu acara.
  3. Pilih anggota tim untuk siapa Anda seperti pemberitahuan yang dikirim.
  4. Pilih Send email to new team members jika Anda ingin anggota tim yang Anda tambahkan di masa depan untuk secara otomatis menerima email bahwa anggota tim lain menerima.

Login Sitemap

JIRA Integrasi memungkinkan Anda untuk berinteraksi dengan JIRA sebagai berikut:

  • Diagnostik Cloud: Buat masalah baru ketika Anda menerima laporan. Unity menciptakan masalah dengan salah satu label berikut: unity-user-report, unity-crash-report, atau unity-exception-report. Anda harus memiliki izin untuk membuat masalah dan memodifikasi kolom label untuk Integrasi ini untuk bekerja.
  • Berkolaborasi: Tambahkan komentar pada masalah yang ada ketika Anda menerbitkan perubahan di Collaborate.

Untuk mengkonfigurasi Integrasi JIRA untuk Diagnostik Collaborate atau Cloud, Anda harus menyediakan informasi berikut:

Parameter Description
Nama tampilan Nama untuk mengidentifikasi Integrasi dalam daftar Integrasi.
Login URL URL dari instance JIRA Anda.
Login Login ID pengguna akun yang dapat memposting pembaruan ke instance JIRA Anda.
JIRA REST API Token Token API untuk mengotenkan permintaan Integrasi ke server JIRA Anda. Untuk instruksi tentang cara membuat token, lihat Atlassian.

Setelah Anda memasukkan informasi untuk instance JIRA Anda, Anda harus menyediakan informasi tambahan berikut untuk diagnostik Cloud:

Parameter Description
Membuat Masalah Dalam Proyek ini Pilih proyek di mana untuk membuat masalah.
Mark Incoming Issues Sebagai Pilih jenis masalah untuk mencatat laporan sebagai.

Ketika Anda melakukan perubahan dalam Collaborate, memperbarui masalah JIRA-associated dengan merujuk kunci isu dalam pesan berkomitmen Anda. Sebagai contoh, “Saya tetap kecelakaan yang disebabkan oleh ISS-42” menambahkan rincian publikasi untuk mengeluarkan “ISS-42”.

Integrasi Trello

Integrasi Unity Trello memiliki fitur berikut:

  • CollaborateLayanan cloud-hosted Unity yang menyediakan cara sederhana untuk tim untuk menyimpan, berbagi, dan berkontribusi pada proyek Unity mereka. More info
    Lihat di Glossary
    • https://trello.com/c/lhLmnCRZ
    • https://trello.com/c/lhLmnCRZ/1-the-card-title
  • Cloud Diagnostics: Ketika Anda menerima laporan pengguna, Unity secara otomatis membuat dan menambahkan kartu baru ke papan Anda.

Untuk mengkonfigurasi Integrasi Trello untuk Collaborate atau Cloud Diagnostics:

  1. Buat Integrasi Trello di Anda. Untuk melakukan ini, pilih Proyek Anda dari dashboard kemudian pergi ke Settings > Integrations dan pilih Trello Integration dari jendela.
  2. Pilih layanan dan acara yang ingin Anda mengintegrasikan Proyek dengan, kemudian pilih Next.
  3. Satuan meminta Anda untuk masuk ke Trello, dan meminta izin untuk membuat perubahan pada papan Trello Anda. Izin ini diperlukan sehingga Unity dapat membuat, memperbarui dan mengomentari kartu.
  4. Isi halaman konfigurasi dengan ringkasan Integrasi, termasuk papan apa dan daftar Unity harus memposting, dan menyimpan pengaturan.

Mengelola Integrasi yang ada

Untuk mengelola yang ada Integrasi:

  1. Masuk ke .
  2. Pilih Proyek dengan Integrasi yang ingin Anda edit.
  3. Di kolom navigasi kiri, pilih Settings > Integrations.

Untuk mengedit atau menghapus Integrasi yang ada, klik EDIT di sebelah Integrasi yang ingin Anda modifikasi. Jenis edit yang dapat Anda lakukan tergantung pada Integrasi:

  • Untuk Integrasi Slack dan Discord, Anda dapat memperbarui nama tampilan atau menghapus Integrasi.
  • Untuk Webhook dan JIRA Integrasi, Anda dapat mengubah parameter konfigurasi yang Anda berikan ketika Anda membuatnya.

Untuk mengaktifkan atau menonaktifkan Integrasi yang ada, klik sakelar Status untuk mengaktifkan atau menonaktifkan Integrasi.
