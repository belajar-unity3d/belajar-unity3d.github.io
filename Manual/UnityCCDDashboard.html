Sitemap Kembangkan Dasbor
Important: Dokumentasi Pengiriman Konten Cloud Unity telah pindah ke . Silakan kunjungi situs baru untuk informasi terbaru tentang CCD Unity.

Anda dapat menggunakan Pengiriman Konten Cloud (CCD) di Kembangkan Dasbor untuk membuat dan mempertahankan ember, lencana dan rilis. Namun, Anda harus menggunakan antarmuka baris perintah (CLI) untuk meng-upload, menghapus dan menyinkronkan file dan folder sebelum Anda mengelolanya di Dasbor Kembangkan.

Sebagai contoh skenario, Anda dapat menggunakan Dasbor Kembangkan secara interaktif mengelola aset aplikasi menggunakan antarmuka pengguna yang lebih visual daripada CLI menyediakan.

Untuk mengakses CCD di Kembangkan Dasbor:

  1. Jika Anda menggunakan layanan Unity untuk pertama kalinya:
    1. Buat akun (jika Anda tidak memiliki satu).
    2. Masuk ke tab Develop.
    3. Klik Create New Project. Dalam Buat Project windowJendela yang menunjukkan isi folder Assets (Project tab) More info
      Lihat di Glossary       Baru, isi detailnya.
  2. Pilih nama proyek, atau klik View.
  3. Di halaman Overview untuk proyek ini, di bilah navigasi sisi kiri, pilih Content Delivery > Buckets.

Buckets

Di halaman Cloud Content Delivery untuk proyek Anda, Anda perlu membuat ember untuk menghosting konten Anda.

Untuk menambahkan ember:

  1. Klik Create Bucket.
  2. Dalam jendela Buat Bucket, berikan ember nama dan deskripsi opsional.

Dalam tampilan daftar ember, setiap ember memiliki baris detail yang berisi informasi berikut:

  • Nama bucket
  • Deskripsi bucket
  • Sebuah snapshot dari keadaan saat ini:
    • Jumlah rilis rilis saat ini, atau pemberitahuan perubahan yang tidak dilepaskan.
    • Tanggal di mana perubahan terakhir dibuat ke ember.
    • Nama pengguna orang yang membuat perubahan terbaru, atau nama sumber untuk perubahan otomatis.

Anda dapat mengedit atau menghapus ember melalui menu More Items (Perusahaan) pada akhir baris ember.

Untuk melihat informasi lebih lanjut tentang ember, klik namanya dalam tampilan daftar. Di layar detail ember, ada tab berikut:

  • Latest: Tampilkan keadaan terkini dari ember, dan versi terbaru dari semua kontennya. Dari sini, Anda dapat membuat rilis. Daftar aset di paginasi, dicari, dan berisi filter. Jika ember Anda tidak memiliki perubahan yang tidak dilepaskan, Anda dapat mempromosikan rilis dari tab ini.
  • Releases: Menampilkan semua rilis yang Anda buat untuk proyek ini.
  • Badges: Daftar semua lencana yang tersedia untuk rilis ini.

Releases

Anda dapat mengelola rilis yang Anda buat untuk proyek Anda pada tab Releases. Jika ember Anda memiliki konten yang tidak dilepaskan, Anda melihat tombol Create Release di baris rincian rilis Anda.

Untuk membuat rilis:

  1. Pilih ember dari halaman Content Delivery Buckets yang mengandung perubahan yang tidak dilepaskan.
  2. Di halaman bucket, pilih tab Latest.
  3. Pilih tombol Create Release.
  4. Dalam kotak dialog Create Release, masukkan catatan rilis opsional dan klik Next.
  5. Di halaman berikutnya, pilih lencana yang ingin Anda aturkan ke rilis ini (menghilangkan mereka dari rilis saat ini), dan pilih Next.
  6. Tinjau perubahan, kemudian pilih Submit untuk menyelesaikan, atau Back untuk terus mengedit. Perubahan pemberitahuan Unreleased Changes untuk mencerminkan status baru.

Setelah Anda membuat rilis, muncul dalam daftar di tab Releases, dengan tanggal pembuatan dan lencana yang terkait. Anda dapat menetapkan lencana untuk rilis melalui menu More Items (Perusahaan) pada akhir baris rilis.

Ketika Anda klik pada Release # rilis dalam daftar, Anda melihat halaman yang berisi rincian rilis, dan daftar aset dan bundel yang terkait dengan rilis itu.

Beberapa situasi di pipa rilis proyek Anda mungkin memerlukan Anda untuk memindahkan rilis dari satu ember ke yang lain, yang merupakan proses yang disebut promosi. Jika ember Anda tidak memiliki perubahan yang tidak dilepaskan, Anda melihat tombol Promote Release di baris rincian rilis Anda.

Untuk mempromosikan rilis:

  1. Klik Release # dari rilis yang ingin Anda promosikan di tab Releases.
  2. Dalam ringkasan rilis, klik Promote Release.
  3. Di jendela Rilis Promosi, pilih ember yang ingin Anda promosikan rilis di drop-down Target Bucket.
  4. Tambahkan catatan rilis opsional, dan pilih Next.
  5. Di halaman berikutnya, pilih lencana yang ingin Anda aturkan ke rilis ini (menghilangkan mereka dari rilis saat ini), dan pilih Next.
  6. Tinjau perubahan, kemudian pilih Promote Release untuk menyelesaikan, atau Back untuk terus mengedit.

Badges

Melampirkan lencana untuk rilis memungkinkan aplikasi Anda untuk memuat konten dari mana pun melepaskan lencana ditugaskan untuk.

Untuk menciptakan lencana:

  1. Pilih ember dari halaman Content Delivery Buckets.
  2. Pilih tab Badges.
  3. Pilih Add New.
  4. Di jendela Add New Badge, berikan lencana nama unik, dan pilih rilisnya dari daftar drop-down.
  5. Pilih Submit.

lencana sekarang muncul dalam daftar lencana di tab Badges. Anda dapat mengedit rilis terkait lencana, atau menghapus lencana melalui menu More Items (Perusahaan) di akhir baris lencana.
