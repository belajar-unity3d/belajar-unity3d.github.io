Mempersiapkan file model Anda untuk ekspor

Faktor-faktor Scaling

Sistem fisika dan pencahayaan Unity mengharapkan meteran 1 di dunia permainan menjadi unit 1 dalam Model fileFile yang berisi data 3D, yang dapat mencakup definisi untuk jaring, tulang, animasi, bahan dan tekstur. More info
Lihat di Glossary impor.

Default untuk paket 3D yang berbeda adalah sebagai berikut:

  • .fbx, .max, .jas = 0.01
  • .3ds = 0.1
  • .mb, .ma, .lxo, .dxf, .blend, .dae = 1

Ketika mengimpor file Model ke Unity dari aplikasi model 3D dengan faktor scaling yang berbeda, Anda dapat mengubah unit file untuk menggunakan skala Unity dengan mengaktifkan opsi Mengkonversi Unit.

Mengoptimalkan file Anda

Sebelum Anda mengimpor file Anda ke Unity, ada sejumlah langkah yang harus Anda ikuti untuk memastikan bahwa file Anda dioptimalkan dengan cara terbaik.

Pikirkan tentang apa yang ingin Anda ekspor: beberapa objek sceneAdegan berisi lingkungan dan menu permainan Anda. Pikirkan setiap file Adegan unik sebagai tingkat yang unik. Di setiap Adegan, Anda menempatkan lingkungan, hambatan, dan dekorasi, pada dasarnya merancang dan membangun permainan Anda dalam potongan-potongan. More info
Lihat di Glossary sangat penting tetapi yang lain mungkin tidak perlu, sehingga Anda dapat mengoptimalkan data dalam Unity dengan hanya menjaga benda-benda penting. Ini membantu menghilangkan data yang tidak diinginkan dari adegan Anda, dan juga berarti Anda dapat menggunakan eksportir praset atau adegan kustom untuk mengotomatiskan atau menyederhanakan proses ekspor.

Mempersiapkan apa yang perlu Anda masukkan

Pertimbangan khusus tabel berikut Anda harus membuat untuk setiap jenis aset untuk bekerja paling optimal dalam Unity:

Object Preparations
Meshes Semua NURBS, NURMS, splines, patch, dan permukaan subdiv harus dikonversi ke poligon (triangulasi atau quadrangulation)
Bake deformers Pastikan Anda bake deformer ke model dalam aplikasi Anda sebelum mengekspor ke format file .fbx. Misalnya, jika Anda ingin mengekspor rig yang kompleks dari Maya, Anda dapat memanggang deformasi ke bobot kulit sebelum Anda mengekspor Model ke .fbx.
Textures Pastikan tekstur dalam aplikasi Anda baik bersumber dari Proyek Unity Anda, atau menyalinnya ke dalam folder yang disebut Textures di dalam Proyek Unity Anda.

Note: Jangan gunakan opsi Embed Media untuk melancarkan tekstur ke file .fbx. Karena Unity harus mengekstrak tekstur sebelum menggunakan mereka, menanamkan mereka memperbesar proyek Anda dan memperlambat proses impor.
Smoothing Jika Anda ingin mengimpor bentuk campuran normal, Anda harus memiliki kelompok halus dalam file .fbx.

Menyiapkan opsi ekspor .fbx

Sebelum Anda mengekspor file .fbx, pastikan Anda menggunakan versi ekspor .fbx terbaru yang mendukung aplikasi model 3D Anda. Ketika Anda mengekspor ke .fbx, perhatikan setiap pengaturan dalam dialog ekspor aplikasi pemodelan 3D Anda sehingga Anda dapat mencocokkan pengaturan impor .fbx di Unity. Kebanyakan eksportir .fbx memungkinkan Anda untuk mengaktifkan atau mengekspor animasi tertentu, camerasKomponen yang menciptakan gambar sudut pandang tertentu di tempat kejadian Anda. Output ditarik ke layar atau ditangkap sebagai tekstur. More info
Lihat di Glossary, dan efek pencahayaan, jadi jika ada yang hilang ketika Anda mengimpornya ke Unity, periksa untuk melihat apakah itu diekspor.

Verifikasi dan mengimpor ke Unity

Sebelum mengimpor file .fbx Anda ke Unity, verifikasi ukuran file ekspor Anda. Lakukan cek sanitasi pada ukuran file (misalnya, untuk melihat apakah lebih besar dari 10KB). Re-import file .fbx Anda kembali ke perangkat lunak pemodelan 3D yang Anda gunakan untuk menghasilkannya (misalnya, ke dalam adegan baru). Periksa untuk memastikan itu adalah apa yang Anda harapkan.

Untuk mengimpor file Anda ke Unity, ikuti petunjuk untuk Importing, perlu diingat bagaimana Anda mengatur opsi ekspor dalam perangkat lunak pemodelan 3D Anda.
