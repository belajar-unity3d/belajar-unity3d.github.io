Mengkonfigurasi untuk Windows Store

Introduction

Panduan ini menjelaskan proses membangun catatan digital dan hubungan yang diperlukan untuk permainan Unity untuk berinteraksi dengan Toko Pembelian In-App. API pembelian Unity IAPUnity In App Purchase
Lihat di Glossary ditargetkan.

Pembelian In-App (IAP) adalah proses transacting uang untuk barang digital. Toko platform memungkinkan pembelian produk mewakili barang digital. Produk-produk ini memiliki Identifier, biasanya dari jenis data string. Produk-produk memiliki Jenis untuk mewakili daya tahan mereka: subscription, consumable (seukuran sedang dibalik), dan non-consumable (sekap dibeli sekali) adalah yang paling umum.

Windows Server

Introduction

Pengembangan Aplikasi Windows menawarkan pengujian IAP Windows lokal dan jarak jauh.

Halaman ini mencakup pengujian lokal dengan emulator dan sistem penagihan simulasi, kemudian pengujian Windows Store yang membatasi visibilitas publikasi aplikasi bagi mereka dengan tautan aplikasi.

NOTE: Panduan ini menargetkan Windows 10 Universal SDK. Target Windows lainnya tersedia.

Getting started

  1. Tulis permainan menerapkan Unity IAP. Lihat Login Login dan .

  2. Jauhkan pengidentifikasi produk game di tangan untuk digunakan di Dasbor Windows Dev Center Microsoft untuk melakukan pengujian Windows Store jarak jauh kemudian.

Uji IAP lokal

Microsoft menawarkan sistem penagihan simulasi, mengizinkan pengujian lokal IAP. Ini menghilangkan kebutuhan untuk mengkonfigurasi apa pun di Windows Dev Center atau berkomunikasi dengan Windows Store melalui aplikasi untuk pengujian integrasi awal.

jauh lebih sederhana daripada untuk jarak jauh Simpan pengujian, meskipun memerlukan perubahan kode sementara ke aplikasi yang perlu dihapus sebelum publikasi aplikasi.

Untuk menguji IAP secara lokal:

  1. Aktifkan sistem penagihan simulasi dalam kode di mana Unity IAP diinisialisasikan dengan instance KonfigurasiBuilder.

    WARNING: Hapus perubahan kode ini setelah pengujian, sebelum dipublikasikan ke Toko; jika aplikasi tidak akan mengubah uang riil melalui IAP!

  2. Membangun aplikasi dalam Unity untuk Universal Windows PlatformFitur IAP yang mendukung simulator Pembelian Aplikasi Microsoft, yang memungkinkan Anda untuk menguji aliran pembelian IAP pada perangkat sebelum menerbitkan aplikasi Anda. More info
    Lihat di Glossary    .

  3. Buka aplikasi di Visual Studio dan jalankan target Mesin Lokal untuk x86.

  4. Uji IAP.

  5. Hapus sistem penagihan simulasi dari kode.

Daftarkan Aplikasi di Windows Store

Setelah fungsi IAP dasar telah diuji secara lokal, Anda dapat lebih percaya diri mulai bekerja dengan Windows Store. Tes ini mengkonfirmasi bahwa aplikasi memiliki semua IAP yang diperlukan terdaftar dengan benar untuk mengizinkan pembelian.

Untuk menguji IAP dan publikasi menggunakan Windows Server dan mengkonfigurasi aplikasi dengan visibilitas terbatas. Ini membatasi visibilitas aplikasi bagi mereka yang memiliki tautan langsung.

NOTE: Pengujian di Toko juga membutuhkan Sertifikasi, yang dapat berfungsi sebagai hambatan untuk pengujian. Oleh karena itu penting untuk menyelesaikan pengujian secara lokal sebelum melanjutkan pengujian dengan Windows Store.

  1. Di Dev Center membuat aplikasi baru.

  2. Cadangan nama aplikasi.

  3. Untuk menguji IAP dengan Windows Store, Windows Dev Center membutuhkan aplikasi yang dipublikasikan. Klik Pricing and availability dan membatasi visibilitas Toko aplikasi sehingga hanya tersedia bagi pengguna yang memiliki link langsung aplikasi.

  4. Dalam “Distribusi dan visibilitas” lihat daftar Toko yang tersedia perilaku publikasi. Pilih Hide this app in the Store.

  5. Kumpulkan tautan langsung. Ini akan digunakan untuk menginstal aplikasi pada perangkat Windows 10 untuk testing.

  6. Kirim aplikasi untuk Sertifikasi.

    Penjelasan dapat memakan waktu banyak jam untuk menyelesaikan, dan memblokir masalah dapat dibesarkan oleh Sertifikasi Microsoft, yang Anda perlu untuk mengatasi sebelum pengajuan melewati berhasil.

Tambahkan Produk In-App di Toko

Tambahkan setiap IAP, atur harga menjadi “gratis” sehingga tidak ada uang yang akan ditransaksikan selama pengujian. Setelah tes selesai, rekonfigurasi IAP dengan harga yang diinginkan dan menerbitkannya. SitemapIAP Submissions.

  1. Di halaman "Appikhtisar" aplikasi baru, klik Create a new IAP.

  2. Masukkan ID produk.

  3. Mengkonfigurasi jenis, harga, dan bahasa.

    NOTE: Untuk Pricing and availability memilih free untuk tujuan pengujian untuk menghindari pengulangan biaya keuangan yang tidak perlu. Ketika Anda selesai dengan pengujian, pembaruan, dan re-submit setiap IAP dengan harga yang diinginkan dalam persiapan untuk rilis ke publik.

    Pilih Properties untuk mengatur jenis.

    Pilih Pricing and availability untuk mengatur harga memilih Free seperti dijelaskan di atas.

    Pilih Manage languages dan deklarasikan bahasa yang didukung.

    Pilih bahasa yang dinyatakan ketika kembali ke ikhtisar IAP.

    Populasi Judul, Deskripsi dan Ikon.

  4. Kirim IAP untuk Sertifikasi.

    Seperti aplikasi, pengajuan IAP dapat memakan banyak jam untuk menyelesaikan, dan memblokir masalah dapat dibesarkan oleh Sertifikasi Microsoft yang Anda perlukan untuk mengatasi sebelum pengiriman melewati berhasil.

Uji IAP dengan Toko

Langkah-langkah ini mengikuti cabang proses uji beta memungkinkan dengan Windows Store. Ini melibatkan membatasi visibilitas aplikasi itu sendiri, mengabaikan penemuan oleh publik melalui fungsi "Pilih Toko". Lihat sumber daya Windows Dev Centre pada Pengujian beta dan distribusi yang ditargetkan untuk informasi lebih lanjut.

  1. Konfirmasikan aplikasi dan IAPs telah menyelesaikan Sertifikasi.

  2. Instal aplikasi pada perangkat Windows 10 melalui link langsung, yang dihasilkan di atas.

  3. Uji IAP.

  4. Setelah lulus tes, memperbarui IAP dengan harga publik yang diinginkan, memperbarui pengaturan visibilitas aplikasi untuk berbagi dengan publik umum, dan menyerahkan kedua jenis perubahan untuk Sertifikasi akhir.
