Buat dan mengelola Dokumen UI (UXML)

Saat ini membuka dan aktif UI(User Interface) Memungkinkan pengguna untuk berinteraksi dengan aplikasi Anda. Unity saat ini mendukung tiga sistem UI. More info
Lihat di Glossary Document (UXML) nama aset ditampilkan dalam pane Hierarchy sebagai item pohon akar serta dalam header Canvas di dalam ViewportThe user’s visible area of an app on their screen.
See in Glossary. Sebuah asterisk * di sebelah nama menunjukkan perubahan yang tidak disadari.

UI Builder akan selalu dimulai, secara default, dengan UI Dokumen yang tidak disaved kosong (UXML). Ini akan ditampilkan sebagai <unsaved file>*.uxml. Anda dapat mulai bekerja dalam Dokumen UI ini (UXML) dan kemudian membuatnya sebagai aset di disk nanti ketika Anda menyimpan untuk pertama kalinya.

Buka Dokumen UI

Anda dapat membuka dokumen UI yang ada dalam beberapa cara:

  1. Gunakan menu File di dalam UI Builder Viewport pane toolbarSe baris tombol dan kontrol dasar di bagian atas Editor Unity yang memungkinkan Anda untuk berinteraksi dengan Editor dengan berbagai cara (misalnya scaling, terjemahan). More info
    Lihat di Glossary     dan pilih Open….
  2. Pada tab Project pane Library, klik kanan pada dokumen dan pilih Open in UI Builder.
  3. Pada Unity Project windowJendela yang menunjukkan isi folder Assets (Project tab) More info
    Lihat di Glossary    , klik dua kali pada dokumen. jika Anda sudah memiliki dokumen terbuka di UI Builder dan klik dua kali di jendela Proyek Unity, UI Document (UXML) akan terbuka menggunakan editor teks default atau IDE.

Anda dapat menyimpan setiap perubahan pada Dokumen UI aktif (UXML) dengan menggunakan menu File > Save di UI Builder's Viewport pane toolbar. Ctrl/CMD + S juga dapat bekerja tetapi jendela UI Builder perlu difokuskan. Perhatikan bahwa Simpan seluruh proyek Unity tidak akan menyimpan Dokumen UI aktif (UXML) terbuka di Pembuat UI.
